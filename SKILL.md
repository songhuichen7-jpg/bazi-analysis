# BaZi 八字命理分析方法论

**依赖**：本 skill 依赖同目录下 `classics/` 文件夹中的五部经典原文（《子平真诠》《滴天髓阐微》《穷通宝鉴》《渊海子平》《三命通会》）。若 `classics/` 缺失，"古籍优先"原则将退化为 LLM 自由发挥——务必保证该目录完整。

**支持文件**（同目录，按需 Read）：
- `classical-references.md` — 古籍位置、何时引、怎么引（做分析前必读）
- `docs/skills/synthesizer-bug-prevention.md` — 8 个实战 bug + 内部 checklist（力量擂台后、白话翻译前内部跑）
- `docs/skills/advanced-techniques.md` — 歌曲匹配、六合化气、五行类象、大运起运（可选）
- `conversation-guide.md` — 用户引导、诗意断词、心灵慰藉、对话节奏（每次对话必读）

**术语速查**（避免不熟悉命理的读者卡壳）：
- **十神**：以日主（出生日天干）为参照点的十种关系（比肩/劫财/食神/伤官/正财/偏财/正官/七杀/正印/偏印）
- **正缘**：命定的稳定伴侣关系（区别于一般桃花）
- **大运**：每 10 年一换的运程阶段
- **流年**：当年的天干地支，与命盘和大运产生交互
- **格局**：以月令为核心定义的命盘"主结构"，决定整体喜忌
- **用神 / 忌神**：对格局有利 / 有害的五行或十神

## §0 输出风格（最高优先级，覆盖所有其他规则）

### 0.1 聊天 > 报告

默认形态：**像一个懂命理的朋友在聊天**，不是审计员在交报告。

❌ 禁止：每次开头汇报"跑了哪些 check"、结尾固定模板（一句话总结+严格边界+反方对抗）、把结论都包成 A/B/C 标签、表格套表格、大量编号走形式。

✅ 例外：用户明确要"完整报告 / 列出所有细节 / 做交叉验证"时才用结构化格式。力量擂台用表格。

### 0.2 大白话 > 专业术语

用术语必须配白话翻译（或先白话后术语）。"你最需要的是表达和创造的出口"比"用神为食伤"好。跟懂命理的人沟通、引用古籍原文时才保留术语。

### 0.3 古籍优先

**有古籍可引时优先引古籍**——没有古籍背书的命理分析本质上是 LLM 在编。

做分析前必须 Read `classical-references.md`。引用流程：Glob `**/classics/[书名]/00_mu-lu.md` → Read 对应章节 → 引原文+白话+reframe+对照命盘。

**🚫 绝对禁止（开源版核心反幻觉规则）**：
1. 凭印象引用但没真的 Read 古籍文件
2. 把 LLM 常识包装成"古籍说……"
3. 编造书名、章节号、原文段落

违反以上任何一条即破坏本 skill 的根本价值。引用古籍前必须实际 Read 对应章节文件。

### 0.4 真边界 vs 防御性回避

**真边界**（绝不能给）：姓名、具体长相五官、具体年月日事件、医疗诊断、财务数字、替命主做选择。

**防御性回避**（大胆给，之前过度限制）：✅ 具体例子和场景 ✅ 应用方法和策略 ✅ 类象气质的具体描述 ✅ 流年方向（标注即可）✅ 大运段特点 ✅ 基于结构的合理猜测（标注"这是基于结构的推测"）

**核心**：有具体信息内容 ≠ 巴纳姆。具体推论大胆给，不确定时先给推论+标注，而不是拒绝。

### 0.5 checklist 与反方对抗：内部跑，不外显

反偏差 checklist 是自己内部跑的，不写在输出里。反方对抗也是——只在它改变了原结论时才在输出中提到。

## 分析入口（收到命盘后按此顺序，不跳步）

1. **Read `classical-references.md`** — 了解可用古籍和查阅路径
2. **排盘验证** — 四柱、十神是否正确（不确定查渊海子平干支体象章）；**未排过的日期必须联网反复核验**（见下方"排盘核验铁律"）
3. **§2 定格局** — 月令三类规则 → 格局名 → 成败判断 → 调候补充
4. **§4 力量擂台** — 各十神透干/得令/根/合克 → 日主强弱 → 合/克/生形态
5. **§5–§7 动态分析** — 身弱方向 / 弱信号识别 / 天干合 / 宫与星 / 合而不化
6. **白话翻译前** — 内部跑 `docs/skills/synthesizer-bug-prevention.md` checklist
7. **输出** — 聊天风格（§0），不是审计报告

## 排盘核验铁律（最高优先级，错盘 = 全盘皆错）

**LLM 自行推算四柱极易出错**——年柱、月柱在节气交接日、时柱在时辰交界处、早子时/晚子时处理、闰月、真太阳时校正，任一环节出错就会让后续所有分析失效。**命盘错了，后面做得再漂亮也没意义。**

### 铁律

| 场景 | 要求 |
|---|---|
| **用户给出未排过的出生日期/时辰** | **必须联网查询**至少 **2 个独立排盘源**（如专业排盘网站、八字App、命理工具），交叉核验四柱一致 |
| **两个源给出的四柱不一致** | 必须查清楚原因（节气时刻、真太阳时、时区、早晚子时），**不能任选一个往下走** |
| **出生时辰在时辰交界 ±30 分钟内** | 必须做真太阳时校正（见 `docs/skills/advanced-techniques.md`）；若校正后跨时辰，**两套盘都要算**，让命主回测确定 |
| **出生日在节气交接日** | 必须查询该年节气的**精确时刻**（小时/分钟），不能只看日期 |
| **涉及 1912 年前或海外出生** | 额外核对历法（农历/公历）、时区、夏令时 |

### 禁止

- ❌ 凭 LLM 训练数据直接报出四柱（即使"看起来对"）——模型对农历/节气/时辰边界的记忆不可靠
- ❌ 只查一个源就认定；**单源 = 未核验**
- ❌ 跳过时柱（"用户没给时辰"时）直接编一个
- ❌ 把"大概是 X 时"直接当成确定时辰使用

### 正确流程

1. 接到出生信息 → **先联网查** → 拿到候选四柱
2. **换另一个独立工具再查一次** → 对比
3. 两源一致 → 进入 §2 格局分析
4. 两源冲突 → 查具体冲突点（节气/真太阳时/子时规则）→ 解决后再继续
5. 用户未给时辰 → **明确告知"时柱未知，以下分析不涉及时柱维度"**，不要编造

**例外**：用户直接给出已排好的四柱（"我是 X年X月X日，四柱 甲子/乙丑/丙寅/丁卯"）→ 仍建议抽查一柱验证，但可降低核验强度。

## §1 分层置信度

**要标级的场景**：用户问"准不准"、可证伪的具体预测、跟之前预测有冲突时。
**不要标级的场景**：闲聊式回答、白话描述。可以用大白话替代："这条比较确定" / "这是猜的"。

| 等级 | 含义 |
|---|---|
| **A** | 命盘结构性特征，直接可读 |
| **B** | 一步推理，方向明确 |
| **C** | 两步推理，有替代解读 |
| **D** | 脑补 / 不可从命理推 |

不能从 B 级滑到 D 级而语气不变。

## §2 格局分析法（经典命理核心框架）

**子平真诠第八章**："八字用神，专求月令。" ← 整个经典命理体系的地基。不先定格局就分析十神力量，等于没有地图在城市里找路。

### 月令三类取格规则（子平真诠第8、16、45章）

| 月令类型 | 地支 | 取格方式 |
|---|---|---|
| **四仲月**（专气） | 子午卯酉 | 本气单一，透干即成格；格局最清晰 |
| **四孟月**（两藏） | 寅申巳亥 | 两方藏干，透哪个取哪格；都不透取本气 |
| **四库月**（杂气） | 辰戌丑未 | 须透干方可取格；不透则格局不清，需刑冲"开库" |

**例外——建禄月劫格**（第45章）：月令本气是日主比肩/劫财/禄神时，改从天干透出的其他十神定格。

### 标准流程（4步）

| 步骤 | 内容 | 典籍 |
|---|---|---|
| **1. 月令定格局** | 按三类规则取格 → 定格局名 | 子平真诠第8-12章、第45章 |
| **2. 判断成败** | 喜用神护格→成；忌仇神破格→败；有救→成中带损 | 第9章 |
| **3. 调候补充** | 日主在该月份的寒暖燥湿喜忌 | 穷通宝鉴·对应日干月份（逐字读） |
| **4. 大运修复** | 大运是否补足格局缺陷 | 第25-26章 |

### 相神（子平真诠第15章）

"相神者，辅我用神之神也…伤相甚于伤用。" 护住用神的十神。相神被破有时比用神被破更致命。

### 用神变化（第10章）

天干三合、六合可改变格局。月令用神被合走 → 须重新认格。化气是否成立须先过§6.1三条件checklist。

### 逆用格与从格

**逆用格**（第19章）：七杀、伤官、枭神、劫财有时反而能成格——此时它们不是忌神，是格局的用神。

**从格**（滴天髓·衰旺/形象章）：日主极弱无根，四柱一边倒时须顺从格局之气。从格时喜忌逻辑**完全翻转**。须查滴天髓衰旺章定性，不能靠旺衰表浅估算。

**铁律**：格局只是框架，身强才能承载。格局与力量擂台独立分析，最后对照。

**严格执行顺序**：先从命盘独立推导 → 再与事实对比 → 不一致时说明原因。

- 推理链超过 2 步
- 换一个相反的事实，你也能找到看似合理的推理链

## §4 力量对比（核心）

评估每个十神的 4 个维度：

| 维度 | 权重 |
|---|---|
| 是否透干 | 高 |
| 是否得令（月令本气） | 最高 |
| 根的数量和质量（地支藏干） | 高 |
| 是否被克/合/冲 | 中 |

**铁律**：任何"X 十神 → Y 行为"的推理，必须先比较 X 与其他十神的相对力量。力量擂台后还必须跑"日主与各十神的合/克/生"分析（`docs/skills/synthesizer-bug-prevention.md` 类型 F）。量决定大小，合/克/生决定形态。

**任铁樵警告**（滴天髓·月令章）："得月令者未必旺，失月令者未必弱"——地支通根数量和质量同样重要，不能死板以月令论旺衰。

## §5 身弱的方向性（最易犯的系统性错误）

**身弱 ≠ 保守/谨慎/内向/不敢**。身弱 = 抵抗力弱 → 被推着走。

| 常见误读 | 正确解读 |
|---|---|
| "身弱→不敢表达" | 身弱→想表达但刹不住 |
| "身弱→保守谨慎" | 身弱→冲动后缺乏持续力 |
| "身弱→内向" | 身弱不决定内外向，看食伤+比劫 |

**§5 是反误读规则，不是普适铁律**。具体表达/自主性必须看食伤+比劫的具体力量。

**从格例外**：符合从格条件时，§5 的所有"身弱"推论完全失效。从格的人不是"被推着走"——而是借力成事，喜忌逻辑完全翻转。

## §5.1 弱信号 = 噪音原则

同一命理因素被不同合格分析者解读出相反方向 → 该因素不具可靠预测价值 → 作为噪音剔除。

| 领域 | 效力 |
|---|---|
| 桃花/神煞判运势吉凶 | **无效**（滴天髓直接否定） |
| 桃花/神煞判外貌相貌 | **有效**（三命通会体系独立权威，与滴天髓属不同体系） |
| 三刑判灾难 | **实质弃用**（子平真诠"姑且阙疑"；滴天髓"冲为重，刑动不动"） |

遇到三刑优先看冲。弱信号因素只作辅助，标低置信度。

## §6 天干六合的动态效果

天干合是命盘中最强的动态交互，遗漏即系统性失真。

| 效应 | 说明 |
|---|---|
| **软化效应** | 合的双方互相"软化"→刚性特质降低 |
| **锁定效应** | 持续、稳定、不衰减的吸引力 |
| **差异化表现** | 面对"合"的对象 vs 其他人→行为完全不同 |
| **化气效应** | 合化成功→日主 identity 根本变化 |

分析正缘/关系动态时，**必须检查日主与正财/偏财之间是否有合**。详细化气分析见 `docs/skills/advanced-techniques.md`。

---

## §6.1 合而不化的判定（实战高频错误）

"见合就化"是极常见的系统性错误。化的条件极严格：

| 条件 | 说明 |
|---|---|
| **月令支持化的五行** | 如庚乙合化金 → 需金旺的月份（申酉月） |
| **四柱有化的五行根基** | 地支需有该五行的根 |
| **无克制化的五行** | 如化金 → 不能有旺火克金 |

三条全部满足 → 真化。任何一条不满足 → 合而不化。

**合而不化时的效果**：两干"情留"（有牵绊），不再各自独立运作；**贪合忘生/克/冲**——合的一方可能忘记对第三方的原有功能。子平真诠原文描述为"得十之二三"的弱化效果（"暗强"是现代命理的reframe说法，非古籍术语）。

**争合与妒合**（子平真诠第5章）：
- **争合**：两干争合同一干 → 合力分散，效果大减，可能合而不成
- **妒合**：第三干介入已有合局 → 原有合局被干扰

见多干指向同一合时，先查第5章，不能假定合成立。

**铁律**：任何天干合，先过三条件checklist，再下化/不化判断。

## §7 命盘层级区分

### 天干与地支的独立性

| 层面 | 天干 | 地支 |
|---|---|---|
| 主导 | 思想/性格/互动/意愿 | 现实/身体/突发事件/环境 |
| 修复难度 | 可通过主观能动性调节 | 需要大运/流年的外部力量 |

**铁律**：不能用天干的"好"去抵消地支的"灾"。分析突发事件时只看地支冲/刑。

### 宫与星的区分

| 概念 | 含义 | 示例 |
|---|---|---|
| **宫** | 地支位置，代表"场域/环境" | 日支 = 夫妻宫 |
| **星** | 十神，代表"人/事" | 正财 = 妻星 |

宫和星可能不在同一柱，两者要分别分析。

**子平真诠第二十四章**：
- "既看妻宫，又看妻星"：宫（日支）与星（妻星十神）须分别独立判断
- "妻透而成局，即坐下无用，亦主内助"：前提是"透而**成局**"——妻星透干且帮助格局成立；若妻星透干但破格（成为忌神），反为凶兆

## §8 双链路独立原则

命盘预测与命主当前行为不一致时，不能用后者否定前者。

**规则**：两条链路冲突时取更短更直接的（命盘链路 > 经验链路）。回测数据用于验证命理体系在此人身上是否有效，不用于修改命盘预测。

## 内部检查（完整内容 → `docs/skills/synthesizer-bug-prevention.md`）

做完力量擂台、翻译成白话前，内部跑以下 checklist（不写在输出里）：

**4 个高频 bug**：
- **类型 A**：正/偏对子混淆（笼统说"X 旺"不分正偏）
- **类型 B**：功能近邻十神混淆（食伤 vs 比劫，叛逆/自主主要是比劫；表达通道还有偏财）
- **类型 C**：主题锚定偏差（被用户提问拉走，忘了命盘底色）
- **类型 F**：合/克维度遗漏（只看量不看质）

**偏差自检**：是不是用户说了X之后才找依据？置信度是否悄悄膨胀？是否只讲了有利的部分？是否一直用同一个框架？

→ 触发任何一条 → 重做该结论，但不要在输出里写"我跑了checklist"。

## 高频错误速查

| 错误 | 修正 |
|---|---|
| 笼统说"X 旺"不分正偏 | 显式比对正/偏对子 |
| 只看力量不看合/克/生 | 跑日主与各十神的合/克/生 |
| 食伤近零 → 无表达/无叛逆 | 叛逆/自主看比劫；**表达通道还有偏财** |
| 用五行属性直接映射行为 | 五行属性 ≠ 行为；十神力量对比才决定（"金=木讷"是错的） |
