# 传播学框架与策略

> 最后更新：2026-04-17
> 核心框架：STEPPS（《疯传》）、引爆点三法则（《引爆点》）、SPIN 框架
> 分析目的：用成熟的传播学理论指导产品的病毒式传播设计

---

## 一、理论框架总览

### 本文档使用三个互补的传播学框架：

| 框架 | 来源 | 核心问题 | 对我们的价值 |
|------|------|---------|------------|
| **STEPPS** | Jonah Berger《疯传》 | 人们为什么分享某些东西？ | 指导内容设计（做什么样的内容会被传播） |
| **引爆点三法则** | Malcolm Gladwell《引爆点》 | 流行是怎么引爆的？ | 指导冷启动策略（什么人、什么时机、什么黏性） |
| **SPIN** | Virality in Social Media 学术研究 | 病毒传播的结构性因素 | 补充对传播基础设施的思考 |

## 二、STEPPS 框架应用

STEPPS 是 Wharton 商学院教授 Jonah Berger 基于大量实验数据提出的六大传播法则。以下逐一分析每个法则如何应用于我们的产品：

### S -- Social Currency（社交货币）

**原理**：人们分享让自己看起来聪明、有品位、有趣的内容。分享行为本质是社交自我展示。

**在命理产品中的应用**：

| 策略 | 具体做法 | 示例 |
|------|---------|------|
| **让用户看起来"懂自己"** | 结果描述要具体、有洞察感 | "你不是选择困难，是看得太透所以懒得选" |
| **提供独特身份标签** | 类似 MBTI 的四字母或 SBTI 的标签 | "春金·破格者" / "水旺·隐士型" |
| **制造"内部知识"感** | 引用命理典籍的片段 | "滴天髓说你这种命格叫'金白水清'" |
| **差异化结果** | 避免多人得到相同结果 | 排盘的天然多样性保证了这一点 |

**SBTI 的实践**："吗喽""死者"等标签让分享者看起来"幽默、自嘲、不装"。
**我们的等价物**：需要设计一套**命理标签体系**，既有命理底蕴又有社交货币属性。

### T -- Triggers（触发器）

**原理**：环境中的线索会提醒人们想到某个产品或话题。频繁的触发 = 频繁的提及。

**在命理产品中的应用**：

| 触发器 | 频率 | 做法 |
|--------|------|------|
| **生日** | 年度 | 生日前推送"新一岁运势解读" |
| **节气** | 每 ~15 天 | 节气更替时推送"本节气对你的影响" |
| **流月** | 月度 | 每月初推送月运势 |
| **社会事件** | 不定期 | 当某个社会热点可以用命理框架解读时 |
| **朋友圈** | 每日 | 当用户看到别人分享测试结果时 |

**关键优势**：八字天然有**节气**这个触发器 -- 24 节气 = 24 次/年的推送机会，这是星座产品（12 星座 x 每月 = 12 次）的 2 倍。

### E -- Emotion（情绪）

**原理**：高唤醒情绪（敬畏、兴奋、愤怒、焦虑）促进分享；低唤醒情绪（满足、悲伤）抑制分享。

**命理产品天然能触发的高唤醒情绪**：

| 情绪 | 触发场景 | 表达方式 |
|------|---------|---------|
| **惊讶 / 敬畏** | "这说的也太准了吧" | 具体、精准的性格描述（非巴纳姆） |
| **好奇** | "我到底是什么命" | 悬念式的展开方式 |
| **释然** | "原来我的困境是有原因的" | 将当前困境外归因于命运周期 |
| **幽默 / 自嘲** | 看到命盘中"不太好"的部分 | 用幽默而非说教的口吻表达 |

**需要避免的情绪**：
- 焦虑（"你命里有灾"）→ 触发合规风险
- 悲伤（"你命不好"）→ 低唤醒，抑制分享
- 无感（巴纳姆式模糊描述）→ 零唤醒，零分享

### P -- Public（公开性）

**原理**：行为越可见，越容易被模仿。"Monkey see, monkey do."

**在命理产品中的应用**：

| 策略 | 做法 |
|------|------|
| **可视化结果卡片** | 生成一张精美的分享图，包含标签 + 核心描述 + 品牌水印 |
| **统一视觉语言** | 所有分享图有统一的视觉风格，一看就知道"这是 XX 产品的" |
| **朋友圈适配** | 分享图尺寸、文字大小适合朋友圈浏览 |
| **分享文案模板** | 提供"发朋友圈的文案"按钮，降低分享的编辑成本 |

**SBTI 的实践**：统一风格的结果截图在朋友圈大量出现 → 视觉统一性 = 品牌识别度。

### P -- Practical Value（实用价值）

**原理**：有用的信息更容易被分享，因为分享有用信息让人觉得自己在"帮助别人"。

**在命理产品中的应用**：

| 策略 | 做法 |
|------|------|
| **给出可操作建议** | 不只说"你事业运好"，而是说"25-27 岁是你的事业爆发期，适合大胆尝试" |
| **时间节点** | 标注具体的大运 / 流年时间节点，让用户可以"照着做" |
| **关系指南** | "你和 XX 五行的人最合"→ 用户会分享给匹配的朋友 |
| **节气提示** | "本节气适合 XX / 不适合 XX"→ 日历级的实用信息 |

### S -- Stories（故事）

**原理**：人们不分享数据，人们分享故事。信息嵌入故事中更容易传播。

**在命理产品中的应用**：

这是我们的**最大优势** -- 八字命盘天然是一个"故事"。

| 故事类型 | 示例 |
|---------|------|
| **命运叙事** | "你的命盘讲的是一个从低谷中破局的故事..." |
| **关系叙事** | "你命中的正缘是一个 XX 类型的人，你们会在 XX 时期相遇..." |
| **成长叙事** | "从 18 岁到 28 岁这十年大运，是你从 XX 到 XX 的蜕变期..." |
| **创造者叙事** | "我们用 Cursor 给自己算命，然后做了这个产品..."（创作故事本身也是传播素材） |

## 三、引爆点三法则应用

### 法则 1：个别人物法则

**三类关键传播者**：

| 角色 | 描述 | 在我们场景中对应的人 |
|------|------|-------------------|
| **联系员** | 社交网络极广，认识各种圈子的人 | 大学同学中的社交达人、多社群活跃者 |
| **内行** | 某领域的专家，别人信任他的推荐 | 身边懂命理/星座的朋友、小红书上关注命理的同龄人 |
| **推销员** | 有天然的说服力和感染力 | 善于安利产品的朋友 |

**冷启动策略（适配无资源、无预算的大学生团队）**：

传统路线（找 KOL）对你们不现实。替代方案：**用话题人物的八字分析做社区内容**。

1. **名人/爱豆八字帖子**：挑选目标用户群体（18-28 岁）关注的话题人物和爱豆，做经过回测验证的八字分析帖子。这些帖子本身就是产品能力的展示，发在小红书/微博/豆瓣等社区
2. **大学社交圈裂变**：作为即将毕业的大学生，校友网络就是天然的种子池。在同学群/校友群分享产品，大学生群体本身就是目标用户
3. **"用 AI Agent 算命"的创作者故事**：你们用 Cursor/Claude Code 算命然后做产品的经历本身就是极好的内容素材，发在 B 站/即刻/V2EX 等开发者社区
4. **让产品自己说话**：如果分享卡片足够有趣，Tier 1 用户会自发传播，不需要 KOL 背书

### 法则 2：附着力因素法则

**信息要有"黏性"** -- 看过一次就忘不掉。

| 黏性策略 | 做法 |
|---------|------|
| **一句话标签** | 每个命盘生成一个记忆度极高的标签 |
| **意外信息** | 在报告中穿插"你可能不知道的自己"式的反直觉发现 |
| **金句** | 引用古籍原文的白话翻译，天然有文学感和权威感 |
| **个人化** | 越具体越有黏性 -- "你在 XX 年 XX 月会感到..."比"你未来运势不错"黏 10 倍 |

### 法则 3：环境威力法则

**传播效果受环境影响极大。选择正确的时机 = 事半功倍。**

| 时机 | 原因 | 内容匹配 |
|------|------|---------|
| **春节前后** | 年度总结 + 新年期许 | "你的 XX 年运势解读" |
| **节气节点** | 文化仪式感 + 变化感知 | "立春/立秋对你的特殊影响" |
| **考试季（6 月/12 月）** | 焦虑高峰 | 事业运/学业运解读 |
| **情人节 / 七夕** | 关系需求高峰 | 合盘/姻缘分析 |
| **秋招季（9-10 月）** | 求职焦虑 | 事业方向建议 |
| **年底** | 总结反思情绪 | 年度命盘复盘 |

**150 人法则**（Dunbar's Number）：有效的口碑传播发生在 150 人以内的紧密社群中。微信群通常 < 500 人，核心互动圈 < 150 人 -- 微信朋友圈正是"150 人法则"的完美实现。

## 四、SPIN 框架补充

SPIN 框架（Spreadability, Propagativity, Integration, Nexus）提供了对传播基础设施的分析视角：

| 维度 | 含义 | 对我们的要求 |
|------|------|------------|
| **Spreadability**（可传播性） | 内容是否易于被复制和分享 | 一键生成分享图 + 分享文案 |
| **Propagativity**（传播性） | 传播渠道的效率 | 微信分享优先（JS-SDK 配置） |
| **Integration**（整合性） | 是否能融入用户已有的社交行为 | 融入"发朋友圈""群聊互动"的日常场景 |
| **Nexus**（连接性） | 是否能链接不同的社群 | 从命理圈 → 心理学圈 → 大众圈的跨圈传播 |

## 五、传播设计实操建议

### 5.1 产品内传播设计

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **命格标签系统** | 每个命盘生成一个有趣的标签（非传统术语） | P0 |
| **分享卡片生成** | 前端 Canvas 渲染精美分享图，含标签 + 一句话 + 品牌 | P0 |
| **一键分享到微信** | 微信 JS-SDK 配置，自定义标题/描述/缩略图 | P0 |
| **合盘/关系对比** | 输入两人生日对比关系（天然裂变机制） | P1 |
| **朋友圈分享文案** | 提供3种不同风格的分享文案供用户选择 | P1 |
| **节气推送** | 每个节气更新内容并推送 | P2 |

### 5.2 产品外传播策略（零预算版）

| 阶段 | 策略 | 具体做法 |
|------|------|---------|
| **冷启动** | 名人八字内容 | 用产品分析 3-5 个目标用户群体关注的爱豆/话题人物的八字，发小红书/微博/豆瓣帖子，展示产品能力 |
| **校园裂变** | 同学群种子 | 在自己的同学群/校友群分享，大学生是核心目标用户 |
| **创作者故事** | 开发者社区 | "用 AI Agent 给自己算命，然后做了一个产品"的故事发 B 站/即刻/V2EX |
| **扩散** | 话题升维 | 将"八字"话题升维为"年轻人的自我探索"社会话题 |
| **沉淀** | 自然裂变 | 分享卡片设计足够好，让 Tier 1 用户自发传播 |

### 5.3 命名策略（传播学视角）

好的产品名应该满足：

| 标准 | 好的例子 | 差的例子 |
|------|---------|---------|
| **一听就懂** | "死了么"→ 立刻知道和死亡有关 | "灵渊 AI"→ 不知道是什么 |
| **自带话题** | "SBTI"→ 自带 MBTI 的联想 | "问天 AI"→ 平淡无奇 |
| **低社交风险** | 幽默/自嘲/中性 | "算命大师"→ 社交尴尬 |
| **利于口头传播** | 2-4 个字/音节 | 长名字难以口口相传 |

**思考方向**：
- "命理版 MBTI" 的认知框架 → 名字是否可以和 MBTI/星座产生联想？
- 自嘲 / 幽默路线 → 降低"算命"的社交尴尬感
- 避免"AI"字样 → 当前"AI XX"已审美疲劳

## 六、传播学视角下的 SBTI vs 死了么 vs 我们的产品

| 维度 | SBTI | 死了么 | 我们的目标 |
|------|------|--------|----------|
| **Social Currency** | 自嘲标签 = 高社交货币 | "我在用死了么" = 话题 | 命格标签 = "懂自己"的社交货币 |
| **Triggers** | 无持续触发器 | 每日签到 | **节气 + 流年 = 高频触发器** |
| **Emotion** | 荒诞 / 幽默 | 恐惧 / 反差 | 惊讶 / 释然 / "被看见" |
| **Public** | 朋友圈截图 | App Store 排名 | 统一风格的分享卡片 |
| **Practical Value** | 无 | "紧急联系人通知" | 具体时间节点 + 可操作建议 |
| **Stories** | 无 | "1500元做出第一名app" | **命盘就是一个故事** + 创作者故事 |

**结论**：SBTI 在 Social Currency 和 Emotion 上极强但缺乏其他维度；死了么在 Emotion 和 Stories 上极强但缺乏留存。我们的产品有机会在 **所有六个维度上都做到中等以上**，这是比任何单一维度极强更可持续的传播策略。
