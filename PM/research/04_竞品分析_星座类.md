# 竞品分析：星座 / 人格测试类产品

> 最后更新：2026-04-17
> 覆盖范围：Co-Star、The Pattern、Sanctuary 等全球标杆产品
> 分析目的：从非命理但同类赛道的成功/失败中提取可复用的经验

---

## 一、Co-Star -- 行业标杆

### 1.1 基本数据

| 维度 | 数据 |
|------|------|
| 上线时间 | 2017 年 |
| 注册用户 | **30M+** |
| 用户画像 | 美国 18-25 岁女性中 **1/4** 下载过 |
| 融资 | 2100 万美元（种子轮 $750K → $5.2M → Series A $15M by Spark Capital） |
| 估计年营收 | $100K-$5M（Owler 估算，未披露） |
| 团队规模 | ~50 人 |

### 1.2 产品核心

**输入**：出生日期 + 时间 + 地点（使用 NASA 数据计算精确星盘）

**输出**：
- 完整出生星盘（natal chart）
- 每日个性化运势推送
- 好友关系对比
- 行星运行实时数据

**核心体验**：**每日推送通知**，以"直接、有点巫、有时候直接骂你"的风格著称。

### 1.3 成功因素深度拆解

#### 因素 1：极致的品牌声音

Co-Star 的推送文案催生了大量 meme，成为社交媒体上的文化现象。示例：
- "Check your ego"
- "Do you play well with others?"
- "Stop waiting for permission"

**为什么有效**：
- 足够短、足够尖锐 → 天然适合截图分享
- 自带争议性 → 引发讨论（"今天 Co-Star 又在骂我了"）
- 个人化幻觉 → 用户觉得"它在说我"（巴纳姆效应 + 个性化推荐的组合）

**对我们的启示**：八字报告中需要有"金句"设计 -- 一句话概括命主特征，风格要有态度。

#### 因素 2：极简设计语言

- **黑白极简**，无圆角卡通，无渐变色
- 刻意回避"创业公司美学"
- 使用不相关的艺术图片而非星座符号

**为什么有效**：在一堆"圆角+柔和色调+卡通"的同类产品中，Co-Star 的视觉一秒就被认出来。

**对我们的启示**：视觉设计需要有辨识度，东方命理可以走水墨/极简东方美学路线，而非"仙侠金光特效"。

#### 因素 3：社交优先设计

Co-Star 本质上是一个**披着星座外衣的社交网络**：
- 添加好友后可以看到彼此的星盘对比
- "今日你们的兼容性"是核心互动点
- 好友越多，打开频率越高

**为什么有效**：社交关系链创造了回访理由 -- 不是为了看自己的运势，而是看"今天我和 XX 配不配"。

**对我们的启示**：考虑加入"合盘"功能 -- 用户可以邀请朋友/伴侣查看关系分析，天然裂变。

#### 因素 4：每日推送制造"Trigger"

STEPPS 框架中的 Trigger 原则 -- Co-Star 每天推送，让产品始终在用户的"心理可用区"。

**对我们的启示**：八字不像星座有每日运势，但可以利用**节气变化、流月运势、大运切换**作为推送节点。

### 1.4 Co-Star 的失败与教训

| 问题 | 具体描述 | 我们如何避免 |
|------|---------|------------|
| **专业准确性差** | 使用 Porphyry 宫位系统（仅 5% 占星师使用），被专业社区批评 | 八字排盘使用双源校验 + 公认的万年历标准 |
| **内容深度不够** | "文字匹配而非真正解盘" -- 从预设片段中拼接 | 典籍直引 + 命盘结构性推导，不用模板拼接 |
| **留存靠社交非内容** | 一旦好友不玩了，自己也不打开 | 内容质量本身要有回访价值 |
| **教育功能缺失** | 用户看了几百天推送，对星座的理解没有加深 | 可选的"命理科普"层，让有兴趣的用户深入 |
| **商业化困难** | 用户习惯免费推送，付费转化率低 | 从一开始就设计好免费/付费边界 |

## 二、The Pattern -- "心理蓝图"路线

### 2.1 基本数据

| 维度 | 数据 |
|------|------|
| 上线时间 | 2017 年 |
| 用户量 | **3.5M+** 活跃用户 |
| 下载量 | 1M+（Google Play） |
| 评分 | 3.8/5（App Store） |
| 融资 | 未公开（Channing Tatum 为投资人之一） |
| 订阅费 | Go Deeper+ $14.99/月 |

### 2.2 产品核心

**差异化定位**：**完全不使用星座术语** -- 把星盘数据翻译成心理学语言。

**三大内容支柱**：
1. **Foundational Patterns**：核心性格特征
2. **Developmental Patterns**：当前成长周期
3. **Relationship Patterns**：与他人的互动模式

**附加功能**：
- Bonds（关系对比）
- Timing / Your Impact（运势周期，纯白话表述）
- Cosmic Climate（集体周期）
- Audio Insights（沉浸式音频体验）

### 2.3 成功因素

#### 因素 1：去术语化

The Pattern 的核心创新是**完全隐藏底层星座逻辑**，只输出"你是一个什么样的人"。用户感受是"这个 app 很懂我"而不是"这个 app 在算我的星座"。

**被描述为 "Myers-Briggs meets Brene Brown"** -- 这个定位比"星座app"有更高的社会认可度。

**对我们的启示**：命理术语（十神、干支、五行等）可以在底层运算，但面向用户的输出应以白话/心理学语言为主。已有 skill 的"§0.2 大白话 > 专业术语"原则与此完全一致。

#### 因素 2：写作品质

The Pattern 被专业占星师赞赏的点不是占星准确性，而是**文字的心理深度**。它的描述"engages with genuine psychological complexity rather than generic platitudes"。

**对我们的启示**：叙事品质是核心壁垒 -- 需要在 prompt engineering 中投入大量精力打磨输出文字。

#### 因素 3：名人效应

Channing Tatum 在 2019 年发了一条反应视频，直接引爆了 The Pattern 的增长。后续 Lena Dunham、Tiffany Haddish 等名人跟进。

**教训**：名人效应可遇不可求，不应作为主要增长策略。SBTI 证明了没有名人也能通过产品本身的传播力引爆。

### 2.4 The Pattern 的问题

| 问题 | 具体描述 |
|------|---------|
| **付费墙争议** | 核心功能逐渐移到付费层，老用户不满 |
| **准确性争议** | 不使用术语 = 无法验证，用户质疑"你到底在算什么" |
| **增长依赖外部** | 名人引爆后自然增长乏力 |
| **社区较弱** | 没有 Co-Star 那样的社交图谱 |

## 三、其他值得关注的产品

### 3.1 Sanctuary（iOS）

- **特点**：真人占星师 + AI 结合
- **商业模式**：$5/次真人咨询 + $25/月会员
- **参考价值**：证明"真人 + AI"混合模式有付费意愿

### 3.2 Hinge（交友 App 的星座功能）

- **特点**：在交友资料中展示星座信息
- **参考价值**：星座/命理可以嵌入其他场景（社交、职场匹配等）

### 3.3 MBTI 测试类产品（16Personalities 等）

- **特点**：93 题测试 → 16 种人格类型 → 极其详细的描述（数千字）
- **用户量**：累计数十亿次测试
- **参考价值**：
  - 证明了"性格分类 + 详细描述"的模式有极强的传播力
  - MBTI 的四维度结构（E/I, S/N, T/F, J/P）可以类比八字的五行/十神结构
  - SBTI 直接寄生于 MBTI 认知

## 四、成功要素提取

综合 Co-Star、The Pattern、MBTI 等产品，提取以下成功要素：

### 4.1 传播相关

| 要素 | 描述 | 适用于我们 |
|------|------|----------|
| **可截图的结果页** | 一眼能看懂、有视觉冲击力 | 命盘标签 + 一句话描述 |
| **社交对比功能** | "我们配不配" | 合盘 / 关系分析 |
| **有态度的表达** | 不温不火 = 无人分享 | "金句"设计 |
| **低参与门槛** | 注册 = 劝退 | 只输入生日，不需注册 |

### 4.2 留存相关

| 要素 | 描述 | 适用于我们 |
|------|------|----------|
| **周期性内容** | 每日/每周/每月推送 | 节气运势 / 流月运势 |
| **社交关系链** | 好友互动 | 合盘功能 |
| **深度探索路径** | 入门 → 进阶 → 专业 | 免费标签 → 付费报告 → 对话 |

### 4.3 变现相关

| 要素 | 描述 | 适用于我们 |
|------|------|----------|
| **免费层足够有价值** | 不是"阉割版"而是"完整但基础" | 免费标签 + 简述有社交价值 |
| **付费 = 更深而非更多** | 深度解析而非数量叠加 | 付费解锁叙事报告 / 专题 |
| **订阅需要持续内容支撑** | 没有新内容 = 退订 | 流年 / 节气更新 |

## 五、对我们产品的架构建议

基于以上分析，建议产品架构为：

```
             ┌─────────────────┐
             │   免费层（传播层）  │
             │   命盘标签 + 金句  │
             │   可截图分享       │
             └────────┬────────┘
                      │ 好奇心驱动
             ┌────────▼────────┐
             │   轻度付费层       │
             │   叙事体完整报告   │
             │   3000-5000 字    │
             └────────┬────────┘
                      │ 具体问题驱动
             ┌────────▼────────┐
             │   对话层（高价值）  │
             │   AI 命理对话     │
             │   流年 / 专题解答  │
             └────────┬────────┘
                      │ 持续关注驱动
             ┌────────▼────────┐
             │   持续层（留存层）  │
             │   节气推送 / 流年  │
             │   大运切换提醒     │
             └─────────────────┘
```

**底层 = 严谨（典籍 + 准确排盘），表层 = 有态度（金句 + 叙事 + 白话）。**

这是 Co-Star 做到了表层但缺底层，The Pattern 做到了文字品质但缺社交，而我们有机会**两者兼得**的核心逻辑。
