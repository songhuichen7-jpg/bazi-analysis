# 竞品分析：爆款 Vibe Coding 项目

> 最后更新：2026-04-17
> 覆盖范围：SBTI、死了么、及其他 vibe coding 爆款项目
> 分析目的：拆解病毒式传播的底层机制，提取可复用的设计原则

---

## 一、SBTI（Silly Big Personality Test）

### 1.1 基本信息

| 维度 | 数据 |
|------|------|
| 全称 | Silly Big Personality Test（傻乎乎的大人格测试） |
| 创建者 | B 站 UP 主"蛆肉儿串儿"（约 11 万粉丝） |
| 上线时间 | 2026 年 4 月 9 日傍晚 |
| 爆发速度 | 上线 → 冲上热搜 = **12 小时** |
| 参与人数 | 约 **50 万人**（首日） |
| 单日阅读量 | **破亿** |
| 开发方式 | Vibe Coding（AI 辅助开发） |
| 网址 | sbti.ai |

### 1.2 产品设计

**输入**：31 道极简选择题，3-5 分钟完成，无需注册

**输出**：27 种人格类型之一，包含以下维度：
- Self Model（自我模型）
- Emotional Model（情绪模型）
- Attitude Model（态度模型）
- Action Drive Model（行动驱动模型）
- Social Model（社交模型）

**标签示例**："吗喽""死者""狗屎人""握草人""尤物""送钱者""伪人"

### 1.3 技术架构

| 层级 | 实现方式 |
|------|---------|
| 前端 | 纯 JavaScript，题目展示 + 逻辑计分 |
| 后端 | **无** -- 完全纯前端计算，零服务器依赖 |
| 数据结构 | 15 维度评分（5 模型 x 3 维度），27 种人格映射全嵌入前端 |
| 结果图 | html2canvas / Canvas API 在浏览器本地生成 |
| 部署 | Cloudflare Workers / 静态托管 |
| 成本 | **极低**（几乎为零，只有域名和 CDN 费用） |

**关键设计决策**：纯前端 = 无限并发能力（CDN 分发），即使服务器"崩"也是 CDN 边缘节点压力，而非后端挂掉。

### 1.4 传播机制拆解

#### 机制 1：极低参与门槛

| Co-Star | MBTI | SBTI |
|---------|------|------|
| 需要精确出生时间和地点 | ~93 题，需要 20-30 分钟 | 31 题，3-5 分钟，不需任何注册 |

**规律**：**参与门槛每降低一级，潜在参与者扩大一个数量级。**

#### 机制 2：自嘲式标签 > 正经分析

MBTI 的标签是 "INFJ -- 提倡者"，Co-Star 是"你的水星在处女座"。

SBTI 的标签是"死者""吗喽""狗屎人"。

**为什么有效**：
- 正经标签分享 = 自夸（社交风险高）
- 自嘲标签分享 = 幽默（社交风险低）
- "他都说自己是 SB 了，我是小丑又何妨" -- 降低分享心理门槛

#### 机制 3：寄生于 MBTI 认知

SBTI 的名字直接模仿 MBTI，用户零学习成本就理解"这是一个人格测试"。

**规律**：**新产品的认知成本越低，传播阻力越小。寄生于已有高认知度的品类是最高效的冷启动方式。**

#### 机制 4：微信生态闭环

```
朋友圈看到截图
     ↓
点击链接测试
     ↓
生成结果图
     ↓
发到自己朋友圈
     ↓
更多朋友看到 → 循环
```

微信的独特优势：
- **熟人信任背书**：来自认识的人的链接天然可信
- **私域到公域闭环**：朋友圈（私域）→ 群聊 → 视频号 / 公众号（公域）→ 再传播
- **半公开剧场**：朋友圈允许筛选观众，降低"暴露"风险

#### 机制 5：崩溃 = 稀缺感

服务器因访问量崩溃 → "我还没测上呢" → 好奇心加倍 → 传播加速。

**规律**：稀缺感是传播的催化剂。但这是意外效果，不可主动设计。

### 1.5 SBTI 的局限

| 问题 | 描述 |
|------|------|
| **零留存** | 一次性测试，没有回访理由 |
| **零商业化** | 完全免费，无变现路径 |
| **无法迭代** | 27 种结果写死，无法生成新内容 |
| **热度衰减快** | 典型的"烟花式"产品，2-3 天后热度骤降 |
| **品效分离** | 获得了巨大的"品"（曝光），但几乎没有"效"（可量化商业价值） |

## 二、"死了么"

### 2.1 基本信息

| 维度 | 数据 |
|------|------|
| 创建者 | 三位 95 后 |
| 开发成本 | **1500 元** |
| 开发时间 | 约 1 个月 |
| 核心功能 | 每日签到确认存活，48 小时未签到通知紧急联系人 |
| 价格 | 8 元（App Store 付费下载） |
| 最高成绩 | App Store **付费榜第 1** |
| 巅峰估值 | 从 1000 万炒到近 **1 亿** |

### 2.2 产品设计

**极简到极致**：
- 无需注册登录
- 三个功能模块：设置紧急联系人邮箱、每日签到、设置姓名
- 连续两天未签到 → 自动向紧急联系人发邮件
- 技术栈：BaaS 平台（Supabase）+ 定时任务 + 邮件服务

### 2.3 传播机制拆解

#### 机制 1：争议性命名

"死了么"直接对标"饿了么"，产生强烈反差。"死"字是禁忌话题，自带猎奇和话题度。

**规律**：**产品名本身就是传播载体。好的名字能把传播成本降为零。**

#### 机制 2：真实痛点 + 黑色幽默

- 中国 1.23 亿独居人口
- "孤独死"恐惧是真实但很少被公开讨论的需求
- 用黑色幽默包装严肃需求 → 降低讨论门槛

#### 机制 3：阶段式传播路径

| 阶段 | 时间 | 渠道 | 特征 |
|------|------|------|------|
| 沉寂期 | 2025.06-2025.12 | App Store | 上架后无人问津 |
| 铺垫期 | 2026.01 初 | 小红书 | 零星 UGC 分享 |
| 引爆期 | 2026.01.11 | 蓝鲸新闻等媒体 | 争议性报道介入 |
| 升维期 | 2026.01.12+ | 全网 | 从"产品"变成"社会议题" |

**规律**：**最强的传播不是"产品好用"，而是"话题值得讨论"。当产品升维为社会议题，传播力指数级增长。**

#### 机制 4：夸张叙事

"1500 元成本""三个 95 后""零营销""App Store 第一名" -- 这些极端数据本身就是传播素材。

**规律**：**产品的创造故事和产品本身一样重要。**

### 2.4 "死了么"的教训

| 教训 | 描述 | 对我们的警示 |
|------|------|------------|
| **技术零壁垒** | 6 小时内被完整复刻（"活了么"） | 产品壁垒必须建立在内容质量而非技术实现上 |
| **无留存设计** | 一次性付费，首周热度后用户不再打开 | 必须有持续性内容钩子 |
| **商业模式脆弱** | 8 元一次性收入，无复购 | 订阅/增值才能覆盖持续成本 |
| **争议反噬** | 被质疑消费死亡恐惧、诱导焦虑 | 命理产品同样面临"迷信""诈骗"质疑 |
| **团队消失** | 改名 Demumu 后从市场消失 | 创始人需要有应对舆论危机的准备 |

## 三、其他 Vibe Coding 爆款参考

### 3.1 数据速览

| 项目 | 创建者 | 爆发速度 | 关键数据 |
|------|--------|---------|---------|
| Pieter Levels 在线游戏 | 独立开发者 | 3 小时开发 | 17 天达 $1M ARR |
| Base44 | AI 网站生成器 | 6 个月 | 25 万用户，被 Wix 以 $80M 收购 |
| Plinq | 非技术创始人 | 45 天 | $456K ARR，1 万+ 用户 |
| TrendFeed | 非技术创始人 | 4 天开发 | 1 个月达 $10K MRR |

### 3.2 共同规律

1. **速度 > 完美**：所有爆款都是快速上线、快速迭代
2. **个人品牌 > 产品品牌**：创作者本身的故事是传播的重要组成部分
3. **AI 降低了执行门槛，但没有降低创意门槛** -- 好的洞察仍然是稀缺资源
4. **纯工具型产品容易被复刻，有"灵魂"的产品不容易** -- 内容品质、品牌声音是护城河

## 四、综合对比：三类竞品给我们的启示

| 维度 | 命理竞品 | 星座竞品 | 爆款项目 | 我们应该做到 |
|------|---------|---------|---------|------------|
| **准确性** | 差 | 中 | N/A | 高（典籍直引 + 双源校验） |
| **表达力** | 差（模板感强） | 强（Co-Star 金句） | 极强（SBTI 标签） | 强（叙事 + 金句 + 白话） |
| **传播力** | 弱 | 强（社交功能） | 极强（低门槛 + 社交货币） | 要设计"截图时刻" + 微信传播 |
| **留存** | 弱 | 中（每日推送） | 极弱（一次性） | 流年/节气/大运持续内容 |
| **商业化** | 中（付费报告） | 弱（免费习惯强） | 极弱 | 分层付费 + 对话变现 |
| **壁垒** | 弱（排盘库可抄） | 中（品牌声音） | 极弱（6 小时复刻） | 强（典籍质量 + 叙事品质） |

## 五、关键行动项

基于以上分析，以下是产品设计阶段必须落实的事项：

1. **设计"截图时刻"**：类似 SBTI 的结果标签，但融入命理元素。例如：给每个命盘一个有态度的"命格标签"（不是"正财格"，而是翻译成有社交货币属性的白话标签）
2. **微信分享优先**：技术上优先实现微信 JS-SDK 分享配置，确保分享卡片的标题和缩略图有吸引力
3. **免费层要有完整价值**：不是"看一半付费看另一半"，而是免费给你一个"有趣的标签和核心解读"，付费给你"完整的人生叙事"
4. **命名需要花大力气**：产品名本身要有传播力和话题性
5. **准备"产品故事"**：你们的故事（用 Cursor/Claude Code 算命 → 做成产品）本身就是很好的传播素材
6. **节气日历内置**：24 节气 + 流年变化 = 24+ 次自然推送机会/年
