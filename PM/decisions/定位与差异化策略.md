## 一、产品定位

### 1.1 一句话定义

> **"用古籍读你的命盘，用故事讲你的人生。"**

- 强调两个核心差异化：**古籍**（vs LLM 编造）和 **故事**（vs 模板报告）
- 暗示了产品的文化厚度和情感温度
- 适合对外传播时使用

### 1.2 定位公式

```
我们是一个 [用命理学框架帮助年轻人认识自己的自我探索工具]
做到了 [引用五部经典原文 + 叙事化表达]
解决的是 [年轻人在迷茫时想获得方向感和确认感]
区别于 [AI 编造的模板报告和强行预测的传统算命]
```

### 1.3 定位三层结构

| 层级 | 面向用户 | 核心主张 | 竞品对比 |
|------|---------|---------|---------|
| **表层** | 大众用户（Tier 1-2） | "有趣的自我发现" | 比 MBTI 更深，比传统算命更轻 |
| **中层** | 付费用户（Tier 2-3） | "被精准理解的体验" | 比 AI 编造更有据，比"大师"更平等 |
| **底层** | 命理爱好者（Tier 4） | "有典籍依据的严谨分析" | 比所有竞品排盘更准、引用更实 |

## 二、核心差异化

### 2.1 差异化矩阵

| 差异化维度 | 竞品现状 | 我们的打法 | 壁垒高度 |
|-----------|---------|----------|---------|
| **准确性** | 排盘频繁出错 | 双引擎交叉验证 + 节气精确校准 | 中（技术可复刻但需要投入） |
| **内容有据** | LLM 通用知识或编造 | 五部经典原文直引 + 出处标注 | **高**（182 文件典籍体系 + 方法论） |
| **表达方式** | 模板报告 / "训人"口吻 | 叙事体 + 白话 + "像朋友聊天" | **高**（prompt engineering + 品牌声音） |
| **传播设计** | 无刻意设计 | STEPPS 框架指导的分享机制 | 中（理念可学但执行需要迭代） |
| **持续价值** | 一次性报告 | 节气 / 流年 / 大运持续内容 | 中（需要持续运营） |

### 2.2 核心壁垒

**最不可复刻的是：典籍体系 + 方法论 + 叙事品质的组合。**

- 典籍原文：需要花大量时间整理和校对（你们已经做了）
- 分析方法论：分层置信度、先推后验、反偏差机制（经过实战验证）
- 叙事品质：conversation-guide + prompt engineering 的持续迭代
- 这三者的结合不是"抄一下代码"就能复刻的

## 三、目标用户策略

### 3.1 首要目标群体

**18-28 岁、城市、受过教育的年轻人**，尤其是：
- 正在经历人生转折的人（求职、跳槽、分手、毕业）
- 对自我认知有好奇心但不想看太严肃的内容
- 活跃在微信朋友圈，习惯分享测试结果

### 3.2 来自真实用户行为的关键发现

基于创始团队的真实使用数据（见 `research/08_典型用户行为分析_基于真实案例.md`）：

- **用户关注什么取决于"此时此刻最 care 的事"** -- 样本用户近期集中在感情方向是因为个人情境，不代表所有用户都以感情为主。换一个正在求职的用户，事业运可能占 80%。
- **产品策略**：不要预设"哪个领域最重要"，而是让用户自己选择深入方向。免费层给全景概览，付费层按专题解锁。

### 3.3 用户获取优先级

```
Phase 1（冷启动）：同学群 + 名人八字帖子 + 开发者社区故事
Phase 2（扩散）：Tier 2 自我探索型 → 作为核心用户和付费主力
Phase 3（裂变）：Tier 1 好奇尝鲜型 → 通过分享机制大规模获取
```

你们自己就是 Tier 3-4 的典型用户，产品的品质标准来自你们自己的使用体验。

## 四、MVP 策略

### 4.1 MVP 范围

| 功能 | 是否包含 | 理由 |
|------|---------|------|
| 输入生日 → 排盘 | 是（P0） | 核心功能 |
| 命格标签 + 一句话描述 | 是（P0） | 免费层的传播载体 |
| 叙事体完整报告 | 是（P0） | 付费层的核心内容 |
| 分享卡片生成 | 是（P0） | 传播的技术基础 |
| 微信分享配置 | 是（P0） | 传播的渠道基础 |
| 自由对话 | **否（P1）** | token 成本不可预测，先验证报告模式 |
| 合盘/关系对比 | **否（P1）** | 裂变功能，但增加开发复杂度 |
| 节气推送 | **否（P2）** | 留存功能，需要后端和推送系统 |
| 小程序版本 | **否（P2）** | 审核周期长，先用 H5 验证 |

### 4.2 MVP 用户旅程

```
用户看到朋友的分享卡片
     ↓
点击链接进入 H5 页面
     ↓
输入出生年月日时（时辰可选）+ 性别
     ↓ [3 秒内]
看到命格标签 + 核心性格简述（免费，~300 字）
     ↓
分享到朋友圈（一键生成分享图）
     ↓ [部分用户]
付费 ¥9.9 解锁完整叙事报告（3000-5000 字）
     ↓ [少数用户]
付费 ¥29.9 解锁专题深度解读（事业/感情/流年）
```

### 4.3 免费与付费的边界

| 层级 | 内容 | 价格 | 设计目标 |
|------|------|------|---------|
| **免费** | 命格标签 + 核心性格 + 可分享的卡片 | 免费 | 传播 |
| **标准** | 叙事体完整报告（性格 + 事业 + 感情 + 年运） | ¥9.9 | 核心收入 |
| **深度** | 专题解析（事业方向 / 感情详解 / 流年运势） | ¥29.9/专题 | 高价值转化 |

**原则**：免费层不是"阉割版"，而是一个完整的、有社交价值的体验。用户不付费也能获得有趣的标签和核心解读，并因此愿意分享。付费是"想要更多"而非"想要完整"。

## 五、传播策略

### 5.1 传播三步曲（零预算、大学生团队版）

| 阶段 | 目标 | 策略 | 时间线 |
|------|------|------|--------|
| **种子期** | 100 个核心用户 | 名人/爱豆八字分析帖子 + 同学群分享 + "AI Agent 算命"创作者故事 | 发布前 2 周 |
| **引爆期** | 1 万+ 用户 | 选择节气节点集中发布 + 分享卡片自然裂变 | 发布日 ± 3 天 |
| **扩散期** | 10 万+ 用户 | 社交裂变 + 话题升维 + 开发者社区故事 | 发布后 1-4 周 |

**关键认知**：你们没有 KOL 资源，但你们有更好的东西 -- **一个真实的、有说服力的创作者故事**（"大学生用 AI Agent 给自己算命，算着算着做了一个产品"），以及**经过回测验证的名人八字分析能力**（这本身就是最有说服力的产品 demo）。

### 5.2 命格标签系统设计方向

这是传播的核心载体，需要满足：

| 要求 | 说明 | 反例 |
|------|------|------|
| **有文化感** | 融入命理元素但不晦涩 | "正财格·偏印透干" ← 太专业 |
| **有态度** | 不温不火 = 无人分享 | "你很善良" ← 太平淡 |
| **可自嘲** | 降低分享心理门槛 | "命运宠儿" ← 分享 = 炫耀 |
| **有辨识度** | 一看就知道来自这个产品 | 通用描述 ← 无品牌关联 |

**思考方向（举例，非最终方案）**：
- "春金·破局者" -- 融合季节 + 五行 + 性格特征
- "水旺·隐世型" -- 五行主导 + 行为模式
- "木火通明" -- 经典命理术语的白话化

### 5.3 发布时机建议

根据引爆点三法则的"环境威力"原则：

| 候选时间 | 环境优势 | 推荐内容 |
|---------|---------|---------|
| **立夏（2026.05.05）** | 节气节点 + 季节转换感 | "你的夏季运势" |
| **端午（2026.05.31）** | 传统节日 + 社交高峰 | "命理版端午签" |
| **秋分（2026.09.22）** | 秋招季焦虑 | 事业方向解读 |
| **冬至（2026.12.21）** | 年终总结情绪 | 年度运势复盘 |

**建议**：选择距离最近的节气作为 soft launch 时间点，避免在非社交高峰期首发。

## 六、商业模型

### 6.1 收入目标

> 目标：cover 掉运营和 token 成本（比较佛系）

| 用户量级 | 月 API 成本 | 需要的付费转化 | 月收入 |
|---------|------------|--------------|--------|
| DAU 1,000 | ~¥500 | 1%（10 人/天 x ¥9.9） | ~¥3,000 |
| DAU 5,000 | ~¥2,500 | 1%（50 人/天 x ¥9.9） | ~¥15,000 |
| DAU 10,000 | ~¥5,000 | 1%（100 人/天 x ¥9.9） | ~¥30,000 |

**结论**：DAU 1000+ 即可覆盖成本。1% 的付费转化率在行业中属于保守估计（命理类产品通常 2-5%）。

### 6.2 成本结构

| 项目 | 月成本 | 备注 |
|------|--------|------|
| 域名 + CDN | ¥50-100 | Cloudflare 免费层基本够用 |
| 数据库 | ¥0-100 | Supabase 免费层 |
| LLM API | ¥500-5000 | MiMo V2 Flash，取决于用户量 |
| 服务器 | ¥0-200 | Serverless 架构，按需付费 |
| **总计** | **¥550-5400** | |

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| **监管打击** | 中 | 高 | 产品定位为"自我探索工具"，避免"预测""改运"措辞；显著免责声明 |
| **被快速复刻** | 高 | 中 | 核心壁垒是典籍体系 + 叙事品质，不可通过抄代码复刻 |
| **排盘出错** | 低 | 高 | 双引擎交叉验证 + 节气精确校准 |
| **AI 幻觉** | 中 | 高 | 典籍检索用规则引擎，LLM 只做表达 |
| **用户无留存** | 中 | 中 | MVP 先验证传播模型，后续迭代加入流年/节气/对话 |
| **微信封杀链接** | 低 | 高 | 避免"诱导分享"设计；准备小程序作为备选 |
| **付费转化低** | 中 | 低 | 成本极低，1% 转化率即可覆盖 |

## 八、简历价值最大化

项目的首要目标之一是"写到简历里"。以下是确保项目简历价值的建议：

### 8.1 可量化指标

| 指标 | 目标 | 简历表述 |
|------|------|---------|
| 用户量 | 10,000+ | "产品上线 X 周获取 XX,XXX 用户" |
| 分享率 | 20%+ | "用户自发分享率达 XX%" |
| 付费转化 | 2%+ | "付费转化率 X%，月收入覆盖运营成本" |
| 技术 | 双引擎排盘 + RAG | "设计并实现了基于典籍 RAG 的 AI 命理分析系统" |
| 传播 | STEPPS 框架指导 | "运用传播学框架（STEPPS）设计产品增长策略" |

### 8.2 故事角度

- **产品角度**："识别到年轻人对命理类自我探索工具的需求，从零设计并上线"
- **技术角度**："将传统命理经典数字化，构建 RAG 管道实现有据可查的 AI 分析"
- **增长角度**："运用 STEPPS 传播学框架设计产品的病毒式传播机制"
- **创业角度**："极低成本（< ¥5,000/月）运营，实现自负盈亏"
