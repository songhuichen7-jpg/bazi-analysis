# 项目评估与目标校准

> 最后更新：2026-04-17
> 本文档对两位创始人的 idea、架构设计和实践做客观评估，并明确项目的核心目标

---

## 一、核心目标重新明确

**首要目标不是赚钱，不是做一个可持续的商业产品。首要目标是：**

> **做一个爆款 -- 哪怕是烟花式的 -- 给两位即将求职的大学生的简历增加一段有说服力的叙事。**

这意味着：
- **传播规模 > 留存率** -- 1 万人看到但次日都走了，依然是简历上的好故事
- **速度 > 完美** -- 花 3 个月做一个完美产品但没人看到，不如花 2 周做一个 80 分的产品引爆一波
- **故事性 > 商业性** -- "两个大学生用 vibe coding 做了一个爆款"比"月收入 5000 元"更有简历叙事价值
- **留存是加分项不是必要条件** -- 有留存最好，没有也不影响核心目标

## 二、客观评估：Idea

### 2.1 好的地方

**1. 找到了一个被验证但未被做好的领域**

120 亿市场、43.7% 增速、3.62 亿曾算过命的人。需求是真的，但现有产品质量普遍低下（排盘不准、AI 味重、巴纳姆效应严重）。这是经典的"巨大市场 + 烂产品 = 巨大机会"的结构。

**2. "命理版 MBTI"的认知框架非常聪明**

MBTI 已经教育了整个年轻人市场"性格测试"的概念。SBTI 直接寄生于 MBTI 认知，零学习成本爆发。"八字版 MBTI"同样可以寄生于这个认知 -- 用户不需要知道什么是"十神""格局"，只需要知道"这是一个关于你的测试"。

**3. 核心团队就是核心用户**

创始人之一本身就是 Tier 3-4 的深度用户，对产品品质有亲身体感。这比任何用户调研都有价值 -- 你知道什么是"准"、什么是"AI 味"、什么会让你"截图发朋友圈"。

**4. 灵感来源本身就是最好的产品故事**

"用 Cursor/Claude Code 给自己算命，算着算着做了一个产品" -- 这个故事在 2026 年的 vibe coding 热潮下，本身就有传播价值。SBTI 的创作者故事（B 站 UP 主、11 万粉丝、为了让朋友戒酒做了个测试）是其传播的重要组成部分。

### 2.2 需要警惕的地方

**1. "命理"比"星座"社交风险更高**

在中国文化语境中，"算命"比"看星座"有更强的负面联想（迷信、骗人）。Co-Star 可以堂而皇之地说"I'm a Virgo"，但中国年轻人可能不太好意思说"我刚算了个八字"。**命名和品牌调性是生死线** -- 需要把"八字"包装成年轻、有趣、去迷信化的体验。

**2. 合规是悬在头上的达摩克利斯之剑**

2026 年 4 月刚出台的《人工智能拟人化互动服务管理暂行办法》+ 2025 年的清朗行动，监管在收紧。虽然定位为"自我探索工具"可以规避大部分风险，但如果产品爆了，被媒体扣上"AI 算命"的帽子，可能面临舆论压力。

**3. 八字比星座天然缺少"每日触发器"**

Co-Star 的杀手锏是每日推送。八字命盘不变，没有"今日运势"的天然触发点。虽然有节气和流年可以做周期性内容，但频率远低于星座。**对于"烟花式爆款"目标，这不是大问题；但如果想做留存，这是核心挑战。**

**4. "深度"和"爆款"之间有张力**

你们的核心壁垒（典籍直引、多 Agent 交叉验证、分层置信度）面向的是 Tier 3-4 用户。但爆款需要的是 Tier 1 用户的大规模参与（3 分钟测试 → 截图分享）。**需要在两者之间找到平衡 -- 表面极简（SBTI 级别的参与门槛），底层极深（典籍级别的内容质量）。**

### 2.3 综合评分

| 维度 | 评分 (1-10) | 说明 |
|------|------------|------|
| 市场机会 | **8** | 需求真实、市场大、竞品差 |
| Idea 独特性 | **7** | "命理镜子"定位好，但 AI 八字不是全新品类 |
| 团队-市场匹配 | **9** | 创始人自己是深度用户，对品质有极高标准 |
| 爆款潜力 | **7** | 有社交货币属性，但"算命"的社交风险需要通过品牌化消解 |
| 技术可行性 | **9** | 所有技术组件（排盘库、LLM、前端分享）都现成 |
| 执行难度 | **6** | MVP 不难，但"命格标签系统"的设计需要创意 |

## 三、客观评估：架构设计与实践

### 3.1 已有资产的评估

| 资产 | 质量评分 | 评估 |
|------|---------|------|
| **bazi-analysis skill** | **9/10** | 182 文件、五部经典、完整方法论 -- 远超任何竞品的典籍覆盖。这是真正的壁垒。 |
| **方法论体系** | **9/10** | 分层置信度、先推后验、反偏差机制、排盘核验铁律 -- 这套方法论经过数百路 Agent 的实战验证，不是纸上谈兵。 |
| **Prompt 迭代** | **8/10** | v1 到 v4 的迭代暴露了所有关键问题（假多 Agent、阶段污染、典籍幻觉等），这些"踩过的坑"直接就是产品的质量保障。 |
| **conversation-guide** | **8/10** | "聊天 > 报告""大白话 > 术语""真边界 vs 防御性回避" -- 这套输出风格规范直接定义了产品的品牌声音。 |

### 3.2 技术架构评估

建议的"双引擎排盘 + 典籍检索规则引擎 + MiMo Flash"架构是务实的：

**好的决策**：
- 前端排盘（隐私保护 + 低延迟 + CDN 扩展性）
- 规则引擎做典籍检索（不依赖 LLM 做检索 = 不会幻觉）
- MiMo Flash 做表达（成本极低 = 覆盖成本无压力）
- H5 先行（跳过小程序审核 = 快速上线）

**需要验证的假设**：
- MiMo V2 Flash 对中文命理术语和古文的理解能力够不够？需要实测
- 前端排盘库（Lunisolar.js / mystilight-8char）的准确性需要和你们自己的排盘结果交叉比对
- "命格标签"的生成逻辑 -- 是用规则映射还是 LLM 生成？规则更可控但创意受限

### 3.3 实践中的问题

**最大的风险不是技术，是"命格标签系统"的设计**。

SBTI 的标签（"吗喽""死者""狗屎人"）之所以传播力强，是因为：
1. 足够意外和好笑
2. 自嘲属性降低分享门槛
3. 每个标签都是独立的"迷你人设"

你们的命格标签需要做到类似的效果，但要融入命理元素。这是一个**创意问题而非技术问题** -- 需要花大量时间 brainstorm 和测试，不能靠 AI 自动生成。

## 四、基于"简历叙事"目标的策略调整

### 4.1 简历上最值钱的数据

面试官关心的不是你用了什么技术栈，而是**规模和影响力**。以下是按价值排序的简历数据：

| 数据 | 简历叙事价值 | 如何获得 |
|------|------------|---------|
| **"XX 万用户"** | 极高 -- 证明产品能力 | 传播设计（免费层 + 分享卡片） |
| **"自发传播率 XX%"** | 高 -- 证明产品设计 | STEPPS 框架指导的分享机制 |
| **"App Store/小红书热搜"** | 极高 -- 社会证明 | 爆款传播 + 话题升维 |
| **"Vibe coding 从 0 到 1"** | 高 -- 2026 年热词 | 创作者故事 PR |
| **"基于 RAG + 五部经典典籍"** | 中 -- 技术深度 | 已有 skill 体系产品化 |
| **"cover 运营成本"** | 中 -- 商业sense | 分层付费设计 |

### 4.2 面向"爆款"的优化

既然烟花式也可以接受，那策略应该是：

**最大化首次传播的爆发力，而不是长期留存。**

具体调整：

1. **MVP 范围进一步精简**：只做"输入生日 → 命格标签 + 分享卡片"，连付费报告都放到 v2。
   - 原因：SBTI 没有任何付费功能照样爆了。付费功能增加开发周期但不增加传播力。
   - 如果 v1 验证了传播力，v2 再加付费报告和对话。

2. **速度是一切**：2 周内上线 MVP，选最近的节气节点发布。
   - 立夏（2026.05.05）是最近的理想发布时间点
   - 或者直接找到下一个有话题性的时间点（毕业季/秋招前）

3. **创作者故事先行**：在产品上线前就开始发"用 AI Agent 算命"的内容（B 站/即刻/V2EX），积累初始关注度。

4. **名人八字帖子先行**：在产品上线前就开始在小红书/微博发话题人物的八字分析帖子，导流到产品。

### 4.3 两条路径

| 路径 | 策略 | 时间线 | 简历叙事 | 风险 |
|------|------|--------|---------|------|
| **快攻路线** | 2 周做纯前端 MVP（命格标签 + 分享卡片），尽快引爆 | 5月中旬上线 | "2周vibe coding做出XX万用户产品" | 内容深度不够，可能被诟病"又一个AI算命" |
| **品质路线** | 4-6 周做完整 MVP（标签 + 报告 + 典籍引用），选节气节点引爆 | 6月中旬上线 | "基于五部经典的AI命理产品，XX万用户" | 更慢，但内容壁垒更高 |

**建议**：走品质路线但保持紧迫感。你们的核心壁垒（典籍体系）是快攻路线放弃的东西，也是让你们区别于"又一个 AI 算命"的东西。但不要追求完美 -- 4 周是上限。

## 五、能不能做成：客观概率评估

### 5.1 先说结论

**有机会，但不是稳赢。成功概率大约在 30-45% 之间 -- 在 vibe coding 项目中已经算高了。**

### 5.2 为什么说有机会

**1. 时机窗口存在**

2026 年的 vibe coding 热潮 + AI 成本暴降 + 年轻人玄学消费上升，三条线交汇。SBTI 证明了"测试类 + 微信传播"的路径是通的，你们要做的不是发明新路径，而是在已验证的路径上换一个载体（八字替代 MBTI）。

**2. 已有资产质量极高**

182 文件典籍体系 + 经过数百路 Agent 实战验证的方法论 -- 这不是两周能从零搭建的东西。大多数 vibe coding 项目的壁垒是零（死了么被 6 小时复刻），你们的内容壁垒是真实的。

**3. 创作者故事有传播力**

"大学生用 AI Agent 给自己算命，做了一个产品" -- 在 vibe coding 叙事热度正高的当下，这个故事本身就值得讲。

**4. 成本结构极其安全**

MiMo Flash 每次 0.002 元，日活 1 万月成本 5000 元。即使免费也烧不了多少钱。"佛系赚钱"的心态在成本这么低的情况下完全可行。

### 5.3 为什么不是稳赢

**1. 爆款的本质是不可预测的**

SBTI 的创作者自己也没预料到会爆。"死了么"沉寂了半年才突然引爆。**传播有大量随机性** -- 你可以最大化概率，但不能保证结果。

**2. "算命"的社交风险比"MBTI测试"高**

SBTI 的标签是"吗喽""死者" -- 自嘲且无负面联想。但"算命"在中国语境中有更强的社会评判。如果产品名或品牌调性没处理好，目标用户可能觉得"分享八字结果有点丢人"。**命名和标签设计是成败关键中的关键。**

**3. "命格标签"是最难设计的部分**

SBTI 的 27 种标签花了大量创意精力。你们需要设计一套融入命理元素、有趣、可自嘲、有辨识度的命格标签。这是**纯创意工作**，AI 帮不了太多。如果标签不够有传播力，产品就只是"又一个 AI 算命"。

**4. 合规不确定性**

虽然定位为"自我探索工具"可以规避大部分风险，但如果产品真的爆了，媒体可能会扣"AI算命"帽子，监管可能介入。这是尾部风险 -- 概率不高但后果严重。

### 5.4 成功的关键条件（按重要性排序）

| 优先级 | 条件 | 控制度 |
|--------|------|--------|
| **1** | 命名要自带传播力（像 SBTI / 死了么） | 完全可控 -- 但需要创意 |
| **2** | 命格标签要有趣、可截图、可自嘲 | 完全可控 -- 但需要创意 |
| **3** | 分享卡片视觉设计要有辨识度 | 完全可控 -- 朋友负责设计 |
| **4** | 上线时机选对节气/热点节点 | 部分可控 |
| **5** | 首批内容（名人八字帖子）要有话题性 | 完全可控 |
| **6** | 微信传播链路技术顺畅 | 完全可控 |
| **7** | 传播链被微信/平台识别和限制 | 不可控 |
| **8** | 运气 -- 是否有意外的传播加速器 | 不可控 |

前 6 个条件都在你们的控制范围内。这已经很好了 -- 大部分项目连前 3 个都做不到。

### 5.5 如果没爆怎么办

即使没有达到"爆款"级别的传播（10 万+用户），只要做到 **1000-5000 用户**，简历叙事仍然成立：

- "从 0 到 1 构建了基于传统典籍 RAG 的 AI 命理产品"
- "运用 STEPPS 传播学框架设计产品增长策略"
- "Vibe coding 全栈开发，2 人团队 4 周上线"
- "典籍数字化：整理并结构化五部命理经典（182 文件）"

这些叙事在技术面试中都有说服力，不完全依赖于用户规模。**爆款是最好的结果，但不是唯一有价值的结果。**

## 六、最终建议

1. **产品名决定一切** -- 花足够多的时间在命名上。一个好名字（像 SBTI、死了么那样自带话题的）可以让传播成本降为零。这是值得花一整周 brainstorm 的事。

2. **命格标签是第二重要的** -- 这是用户截图分享的核心内容。需要好笑、有态度、融入命理元素但不晦涩。

3. **"用 AI Agent 给自己算命"的故事是你们最大的非产品资产** -- 在产品上线前就开始讲这个故事。

4. **不要被"做一个完整产品"的思维绑架** -- SBTI 就是一个纯前端的 31 题测试，但它的传播力超过了任何"功能完整"的命理 App。

5. **简历叙事的视角应该反过来指导产品决策** -- 每个功能决策都问"这个功能对简历叙事有帮助吗？"如果没有，砍掉。

6. **即使没爆也不亏** -- 最坏的情况，你们也拥有了一个有深度的技术项目 + 一套完整的市场分析框架 + vibe coding 实战经验。这些在面试中都有叙事价值。
