# 标签文案生成系统

> 文档版本：v3.0 | 2026-04-18
> 本文档定义：类型标签和卡片文案的 AI 生成规则、few-shot prompt、风格规范
> **依赖**：`02a_格局标签系统.md`（格局标签命名、金句与解读口径；与 20 型底色联动的可见组合约 200 种）

---

## 一、为什么用 AI 生成而不是手写

- 20 个基础类型 × 多维度文案（标签/金句/描述/子标签）= 大量文案工作；叠加格局标签后可见组合约 200 种（含卡片双层标题）
- 合盘矩阵 20×20，两人无序组合 210 种 × 每种需要标签+描述 = 更大量
- 手写容易风格不统一
- AI 生成 + 人工筛选 = 效率 × 质量

**工作流**：
```
定义风格规范 → 写 few-shot 示例 → LLM 批量生成候选 → 人工挑选/修改 → 定稿
```

## 二、风格规范（适用于所有文案）

### 基调

```
精准重构 × 场景化自嘲 × 不装不端
```

### 不是什么

| 不是 | 为什么不行 | 反例 |
|------|----------|------|
| 心理测评 | 太学术，无分享欲 | "你具有较强的内省倾向" |
| 算命先生 | 太玄学，年轻人怕被嘲笑 | "你命犯七杀，主有灾厄" |
| 励志鸡汤 | 太正确，无记忆点 | "相信自己，你一定可以" |
| SBTI式骂人 | 太激进，品牌调性不匹配 | "你就是个废物" |

### 是什么

```
是一个很懂你的朋友，用你自己可能说不出口但心里认同的话，
帮你把"以为是缺点的东西"翻面给你看 -- "原来这不是缺点，这是我的底色"。
```

### 文案公式

```
[场景化描述] + [反直觉重构] = 传播力

示例：
"灵感来了挡不住，没来你也催不来"
 ↑ 场景（很多人有过的经历）  ↑ 重构（拖延不是懒，是灵感型）
```

### 语言规则

| 规则 | 标准 |
|------|------|
| 句长 | 单句 ≤20 字 |
| 人称 | "你""我"交替，像在自言自语 |
| 术语 | 零命理术语 |
| 标点 | 允许逗号、句号、省略号、破折号。不用感叹号连用 |
| 语气 | 陈述 > 反问 > 祈使。不用"你应该""你需要" |
| 自嘲 | 允许轻度自嘲，但必须落在"其实这样也挺好"的重构上 |

## 三、类型标签 Few-Shot Prompt

### Prompt：生成类型人格标签

```
你是一位精通中国传统命理和现代传播学的文案专家。

你要为一个八字命理产品"有时"生成类型标签文案。产品面向 18-30 岁的中国年轻人。

每个类型基于一个"天干"（日主）及 **绽放/蓄力** 二元气质，有一个来自古籍《渊海子平》的"底色名"，命名规范形如 **「甲·绽放·参天木命」**（天干·气质·底色名）；卡片上还可能叠加格局行，格局标签文案以 `02a_格局标签系统.md` 为准（如 **「食神格 · 天生享乐家」**）。
你需要为每个 **底色类型**（在已知格局标签仅作语气参考、不替代子平推理的前提下）生成：
1. 人格标签（≤5字，有梗感但不攻击人）
2. 一句话描述（≤20字，场景化 + 反直觉重构）
3. 三个子标签（各≤6字）
4. 金句（2句话，可独立截图分享）
5. 正面重构句（"这不是XX，是XX"格式）

## 风格要求
- 像用户自己说的话，不像产品在介绍用户
- 有画面感，能让人回忆起具体场景
- 轻度自嘲但最终落在"其实挺好的"上
- 不使用任何命理术语
- 不是纯鸡汤正能量

## 示例（作为风格参照）

### 示例 1：丁·绽放·烛灯火命（丁火·绽放）
- 人格标签：闪念型
- 一句话：灵感来了挡不住，没来你也催不来
- 子标签：脑子比计划快 / 夜猫子基因 / 不按套路出牌
- 金句："不是我不专注。是我的专注力自带随机传送门，你不知道它下一秒飞去哪。"
- 重构句：这不是三分钟热度，是灵感有自己的时刻表。
- （卡片双层示意，第二层引自 02a）格局行：食神格 · 天生享乐家

### 示例 2：己·蓄力·田园土命（己土·蓄力）
- 人格标签：慢炖型
- 一句话：急什么，我在种地
- 子标签：后劲最大 / 不争一时 / 慢工出好活
- 金句："你别催我。我只是需要比你多一点时间。但最后到的那个人，一定是我。"
- 重构句：这不是慢，是在攒一股大的。
- 格局行：正官格 · 秩序收纳者（蓄力版示例，具体以月令为准）

### 示例 3：辛·绽放·珠玉金命（辛金·绽放）
- 人格标签：精装型
- 一句话：环境对了就发光，不对就先收着
- 子标签：挑平台 / 低调蓄力 / 对的时机才出手
- 金句："不是我不努力。是我需要对的土壤，才愿意把自己种下去。"
- 重构句：这不是挑剔，是珠玉知道自己值什么。
- 格局行：伤官格 · 锋芒外放者（示例）

## 现在请生成以下类型的文案：

{天干名称}，气质为「{绽放|蓄力}」，完整类型名为「{天干·气质·底色名}」。
格局标签（仅作卡片第二层参考，格局名与产品标签以 02a 定稿为准）：「{十神格·气质版：产品标签}」
《渊海子平》原文："{体象诗原文}"
《三命通会》论其性情："{性情描述}"

请严格按照示例格式输出。
```

## 四、合盘标签 Few-Shot Prompt

### Prompt：生成合盘关系标签

```
你是一位精通命理和社交文案的专家。

为"有时"产品生成合盘关系标签。合盘是两个人的 **20型底色**（天干×绽放/蓄力）组合，共 210 种无序组合；若产品展示双层卡片，可在描述中轻点格局标签差异，但不改变合盘「底色互动」主线。
所有合盘结果必须是正面的。没有"不合"，只有"不同方式的合"。

每个合盘需要：
1. 关系标签（≤4字）
2. 三个子标签（各≤8字）
3. A 的角色描述（≤8字）
4. B 的角色描述（≤8字）
5. 关系描述（2句话）
6. CTA 金句（1句，对话感，可以@对方）

## 风格要求
- 像两个朋友互相吐槽但很温暖
- 强调互补或共鸣，不强调匹配度
- CTA 金句要让人想转发给对方

## 五行关系参照
- 相生：一方给另一方能量 → 滋养型关系
- 相克：有碰撞 → 火花型关系（正面重构：磨砺/碰撞出新东西）
- 相合：天干合 → 默契型关系（最特别）
- 相同：同五行 → 镜像型关系
- 无直接关系 → 互补型关系

## 示例

### 示例 1：甲·绽放·参天木命 × 丁·绽放·烛灯火命（木生火 → 滋养型）
- 关系标签：点火搭子
- 子标签：你是燃料我是火 / 安静地给对方能量 / 一点就着
- A（甲·绽放·参天木）：你的存在就是能量
- B（丁·绽放·烛灯火）：接到就能亮
- 描述："你不需要做什么，你在那里，TA 就能找到灵感。你们之间的能量传递，安静但有效。"
- CTA："有你在我就有电。"

### 示例 2：戊·蓄力·城墙土命 × 壬·绽放·江河水命（土克水 → 火花型/重构为引导）
- 关系标签：河堤搭子
- 子标签：你定方向我冲锋 / 有边界才有力量 / 不是拦你是导你
- A（戊·蓄力·城墙土）：立在那里不动摇
- B（壬·绽放·江河水）：绕不过就冲过去
- 描述："一个是拦不住的水，一个是搬不动的墙。听起来要打架？不，是水终于有了方向。"
- CTA："别拦我。好吧，你拦了我反而知道往哪冲了。"

## 现在请生成以下合盘的文案：

A 类型：{A完整类型名，如甲·绽放·参天木命}（{A天干}·{A五行}）
B 类型：{B完整类型名}（{B天干}·{B五行}）
五行关系：{相生/相克/相合/相同/无直接关系}
{如果是天干合，注明具体哪种合}

请严格按照示例格式输出。
```

## 五、专栏报告 Few-Shot Prompt

### Prompt：生成命格故事报告

```
你是一位精通子平八字命理的分析师，为"有时"产品生成专栏报告。

用户信息：
- 四柱：{四柱}
- 日主：{天干}（{底色名}）
- 底色类型：{天干·绽放/蓄力·底色名}
- 格局标签（十神×绽放/蓄力，文案口径见 02a）：{格局标签完整名}
- 子平格局（月令）：{格局}
- 纳音：{纳音}

可引用的典籍段落：
{渊海子平·干支体象·该天干部分}
{三命通会·卷七·论性情相貌·该五行部分}
{三命通会·卷四·论十干坐支·该日柱部分}
{渊海子平·论性情·该五行部分}

## 报告结构

### 一、你的底色
用 2-3 句话描述这个天干的核心意象，并 **自然融入格局标签（十神绽放/蓄力）带来的个性转折**（与 `02a` 一致，不重复堆砌术语）。引用渊海子平原文并白话化。

### 二、古籍怎么说
引用 2-3 段典籍原文，每段后用 1-2 句白话解读。
解读风格：像在翻译给朋友听，不是在做学术注释。

### 三、你的天然优势
列出 3-5 个优势标签，每个附 1 句场景化描述。

### 四、你可能的盲区
列出 1-2 个盲区，但用正面重构表达：
不说"你容易急躁"，说"你做事很急，不是因为毛躁，是因为想到就要行动"

### 五、在别人眼中
分"初识"和"深交"两个阶段描述。

### 六、给你的一句话
一句金句，可以截图分享。

## 风格要求
- 全文用"你"
- 引用原文时用引号标注出处
- 白话解读不超过 20 字/句
- 总字数 800-1200 字
- 语气温暖但不鸡汤
```

## 六、质量控制流程

### 生成 → 筛选 → 定稿

```
Step 1：用 prompt 批量生成
  - 每个基础类型生成 3-5 套候选文案；格局标签层以 02a 为单一事实来源，必要时只做 A/B 微调
  - 每个合盘无序组合（210 种）抽样或热点组合生成 2-3 套候选

Step 2：初筛（自动）
  - 检查字数是否符合限制
  - 检查是否包含命理术语（如包含则剔除）
  - 检查是否包含负面词汇

Step 3：人工精选
  - 两人各自独立打分（1-5 分）
  - 取交集（双方都≥4 分的保留）
  - 对分歧项讨论修改

Step 4：用户测试
  - 找 5-10 个目标用户
  - 逐个展示结果，问"这说的是你吗"
  - 收集自然反应（是否想分享、配什么文字）

Step 5：定稿入库
  - 固化为 JSON 数据
  - 卡片系统直接读取
```
