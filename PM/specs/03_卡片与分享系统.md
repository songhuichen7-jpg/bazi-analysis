## 二、个人卡片

### 设计原则

用户拿到卡片，**第一眼看到的是传播名**（社交货币），不是命理术语。命理术语（参天木命、绽放型等）保留在详情页/AI报告中。

### 内容结构

```
┌───────────────────────────────┐
│ 查八字                  03 / 20 │  ← 头部：品牌名 + 类型编号
│                               │
│  [插画]                       │
│                               │
│         萨摩耶                 │  ← 传播名（最大字，社交货币）
│      · 天生享乐家 ·            │  ← 十神后缀（小字副标题，02a）
│                               │
│    看着随和，底线焊死            │  ← 一句话（02 基础类型）
│                               │
│  ┌──────┐ ┌──────┐ ┌───────┐ │
│  │随和小 │ │吃喝玩 │ │快乐是  │ │  ← 三个子标签（十神干预）
│  │社牛   │ │乐一把 │ │正经事  │ │
│  └──────┘ └──────┘ └───────┘ │
│                               │
│  " 我不卷，但我什么都不缺。      │  ← 金句（02a 格局标签系统，1条）
│                               │
│  查八字 · chabazi.com          │  ← 底部品牌
└───────────────────────────────┘
```

**与旧版对比**：去掉了「参天木·绽放型」命理术语大字、底色名色块、重构句/第二金句、icon对。卡片更干净，信息更聚焦。

### 200 种独特组合

每张卡片由 **传播名（20种）× 十神后缀（10种）= 200 种** 独特组合决定：

- 传播名决定「部落认同」——"你也是萨摩耶？"
- 十神后缀决定「个人差异化」——"但我是天生享乐家型的萨摩耶"
- 子标签由十神干预，同一传播名不同格局的子标签完全不同

示例（萨摩耶 × 不同十神）：

| 格局 | 后缀 | 子标签 |
|------|------|--------|
| 食神格 | 天生享乐家 | 随和小社牛 / 吃喝玩乐一把手 / 快乐是正经事 |
| 七杀格 | 压力玩家 | 笑着下狠手 / 外柔内狠 / 温柔刀高手 |
| 正印格 | 隐形学霸 | 随和学霸 / 看着玩其实在学 / 知识软着陆 |

### 数据来源

| 区域 | 来源 | 生成方式 |
|------|------|---------|
| 类型编号 | 映射表（20 型：日主天干 × 绽放/蓄力 → 01-20） | 确定性查表 |
| 插画 | 20 张预制插画（传播名意象） | AI 生成 |
| **传播名**（大字） | 02 八字人格类型系统 · 传播名列 | 预定义 JSON |
| **十神后缀**（小字） | 02a 格局标签系统 · 后缀名 | 月令查格局 → 后缀 |
| 一句话 | 02 八字人格类型系统 · 一句话列 | 预定义 JSON |
| **子标签（3个）** | 02c 子标签矩阵（200组） | 预定义 JSON（传播名 × 十神） |
| **金句**（1条） | 02a 格局标签系统 | 预定义 JSON（随格局变） |
| 底部品牌 | 固定 | 配置项 |

**所有卡片文案预写定稿后固化为 JSON**，无需实时 LLM 生成。零 token 成本，毫秒级响应。

### 个性化维度

| 维度 | 个性化点 | 来源 | 组合数 |
|------|----------|------|--------|
| **传播名** | 20种（春笋/橡子/萨摩耶...） | 日主天干 × 绽/蓄 | 20 |
| **十神后缀** | 10种（天生享乐家/压力玩家...） | 月令查格局 | 10 |
| **子标签** | 200组（传播名 × 十神交叉） | 预生成矩阵 | 200 |
| **金句** | 200条（格局 × 绽/蓄） | 02a | 200 |
| **用户昵称** | 可选显示 | 用户输入 | — |

### 技术规格

| 规格 | 值 | 理由 |
|------|---|------|
| 尺寸比例 | 3:4（竖版） | 朋友圈最佳比例 |
| 分辨率 | 1080×1440px（@2x） | 高清不糊 |
| 格式 | PNG | 色彩保真 |
| 生成方式 | html2canvas / dom-to-image | 客户端生成，零服务端压力 |
| 品牌水印 | 底部固定 | 裂变回流保障 |

## 三、合盘卡片

### 内容结构

```
┌───────────────────────────────┐
│ 查八字合盘图鉴                   │
│    04/20 × 06/20    ⚡🔋       │  ← 双方类型编号 + 动态状态（见下）
│                                │
│  (头像A) @小满 × @阿青 (头像B)   │
│                                │
│        ┌─────────┐             │
│        │ 点火搭子 │             │  ← 合盘关系标签
│        └─────────┘             │
│                                │
│  ┌────────┐ ┌──────┐ ┌─────┐  │
│  │你是燃料 │ │安静地给│ │一点  │  │  ← 关系子标签
│  │我是火   │ │对方能量│ │就着  │  │
│  └────────┘ └──────┘ └─────┘  │
│                                │
│   丁火·绽放型    己土·蓄力型    │  ← 双方基础类型（可与昵称行合并展示）
│   灵光一闪       接住就能亮      │  ← 各自角色
│                                │
│  你不需要做什么，你在那里，       │  ← 关系描述
│  TA 就能找到灵感。              │
│                                │
│  有你在我就有电。               │  ← CTA 金句
│  @阿青 看看我们是不是这种搭子    │  ← 分享引导
│                                │
│  查八字 · chabazi.com           │
└───────────────────────────────┘
```

**文案组合规模**：无序类型对共 **210** 种（`C(20,2)+20`），在 **55** 组天干关系骨架上由 **04b** 叠加「绽放/蓄力」修饰模板生成差异化关系描述（与 **04a** 骨架、**04** 合盘系统配合）。卡片头部 **⚡ / 🔋** 组合与四种动态状态一一对应：**⚡⚡** 双绽、**⚡🔋** A 绽 B 蓄、**🔋⚡** A 蓄 B 绽、**🔋🔋** 双蓄。

### 数据来源

| 区域 | 来源 | 生成方式 |
|------|------|---------|
| 双方类型编号 | 各自排盘（20 型） | 确定性 |
| **动态状态标识** | 双方绽/蓄 | **⚡⚡** 双绽放、**⚡🔋** A 绽 B 蓄、**🔋⚡** A 蓄 B 绽、**🔋🔋** 双蓄（定义见 **04b**） |
| 双方昵称 | 用户输入 | 邀请方填写 + 对方输入 |
| 关系标签 + 子标签 + 角色描述 | **55 组天干关系骨架**（04a / 04 合盘矩阵） | 预定义 JSON 查表 |
| **关系描述 + CTA** | 骨架文案 + **04b 修饰模板** | 同序对组合共 **210 种**唯一「类型对 + 绽蓄状态」产出；修饰句叠在骨架后 |
| 插画 | 双人互动插画 | AI 生成（优先按关系大类与天干意象，不必 210 张一图） |

### 合盘插画策略

插画全部 AI 生成。**合盘文案层为 210 种**（`C(20,2)+20`，由 **04b 合盘绽放蓄力修饰系统**在 55 骨架上生成），插画可继续按关系大类 + 天干组合配置，不强求 210 张独立成图。

**方案**：
- 按关系类型分（天作/滋养/碰撞/默契/互补/镜像/同频）→ 6 大类
- 每类定义统一的 AI 生成 prompt 模板（构图/元素/氛围）
- 每种组合传入具体五行元素 → AI 生成对应插画
- 绽/蓄可通过人物动势、光感强弱等轻量变体区分（可选）
- 统一风格：通过 prompt 中的 style reference 保证视觉一致性

## 五、分享链路设计

### 核心原则

```
从看到结果到完成分享 ≤ 2 次点击
```

### 分享路径

```
结果页面
   │
   ├── [保存到相册] ← 1 click
   │      └── html2canvas 生成 PNG → 保存到手机 → 用户发朋友圈
   │
   ├── [分享给好友] ← 1 click（微信内）
   │      └── 微信 JS-SDK → 分享卡片（标题+描述+缩略图+链接）
   │
   └── [邀请好友合盘] ← 1 click
          └── 生成合盘邀请链接 → 直接发给好友
```

### 微信分享配置

| 字段 | 内容 | 示例 |
|------|------|------|
| **title** | 传播名 + 勾起好奇 | "我是萨摩耶·天生享乐家 -- 你是什么？" |
| **desc** | 一句话 + 引导 | "查八字人格图鉴，3 秒看到你的类型" |
| **link** | 带来源标记 | `chabazi.com/?from=share&type=04` |
| **imgUrl** | 类型小 icon / 品牌 logo | — |

### 保存到相册

```javascript
const cardElement = document.getElementById('result-card');
const canvas = await html2canvas(cardElement, {
  scale: 2,
  useCORS: true,
  backgroundColor: '#ffffff'
});
const dataUrl = canvas.toDataURL('image/png');
// 移动端：显示图片 → 引导长按保存（iOS Safari 限制自动下载）
// 桌面端：自动下载
```

### 分享追踪

| 追踪点 | 实现 | 指标 |
|--------|------|------|
| 分享动作 | 按钮埋点 | 分享率 |
| 分享来源 | URL 参数 `from=share&type=XX`（20 型编号） | 各类型分享量 |
| 回流 | 新用户带 `from` 参数 | K 值 |
| 合盘邀请 | 链接生成埋点 | 合盘发起率 |
| 合盘完成 | 被邀请方完成输入 | 合盘完成率 |

### 防封策略

| 红线 | 应对 |
|------|------|
| "分享后解锁" | **绝对不做** |
| 诱导转发 | 分享按钮是"工具"不是"条件" |
| 频繁触发 JS-SDK | 控制调用频率 |
| 域名被封 | 备用域名 + 短链 |
| 小程序限制 | 如做小程序版走小程序卡片分享 |

## 六、视觉设计规范

### 整体风格

```
温暖手绘感 × 现代排版 × 留白呼吸感
```

- **配色**：暖色调为主，每型（天干 × 绽/蓄）有主题色锚点
- **插画**：手绘线条，人物简笔有辨识度
- **字体**：标题圆体/手写体，正文无衬线
- **圆角**：12-16px

### 20 型主题色（10 天干底色 × 绽放/蓄力）

配色以 **天干** 为主线：同一干支意象的绽放 / 蓄力共用同一色相，靠 **明度、饱和度或辅助点缀** 区分（例如绽放略亮、蓄力略沉）。下列为天干主色锚点；正式编号与命名以 **02** 为准。

| 天干 | 意象锚点 | 主题色 | 色值 | 理由 |
|------|----------|--------|------|------|
| 甲 | 参天木 | 森林绿 | #2D6A4F | 参天木 → 深绿 |
| 乙 | 花草木 | 草木绿 | #52B788 | 花草 → 嫩绿 |
| 丙 | 太阳火 | 阳光橙 | #F5A623 | 太阳 → 明亮橙 |
| 丁 | 烛灯火 | 烛光蓝 | #4A9BE8 | 烛火灵动 → 清亮蓝 |
| 戊 | 城墙土 | 大地棕 | #A0785A | 城墙 → 沉稳棕 |
| 己 | 田园土 | 田园黄 | #D4A574 | 田园土 → 暖黄 |
| 庚 | 刀剑金 | 钢蓝 | #4A7BA8 | 刀剑金 → 冷蓝 |
| 辛 | 珠玉金 | 薰衣草紫 | #9B7AC4 | 珠玉 → 温柔紫 |
| 壬 | 江河水 | 深海蓝 | #1A759F | 江河 → 深蓝 |
| 癸 | 雨露水 | 薄荷蓝 | #4AC4C0 | 雨露 → 清透蓝绿 |

以上为建议色值，最终由设计师定稿。

## 七、质检清单

每张卡片定稿前过一遍：

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 3 秒可读 | 扫一眼就知道类型 + 大意 |
| 2 | 截图可分享 | 不需额外解释 |
| 3 | 产品入口 | 底部有品牌名 + 网址/二维码 |
| 4 | 无裸命理术语 | 不出现「日主」「五行生克」「参天木·绽放型」等命理词；卡片正面仅展示传播名 + 十神后缀（如「萨摩耶 · 天生享乐家」），命理详情留在报告页 |
| 5 | 无负面内容 | 全部正面或正面重构 |
| 6 | 字数合规 | 标签≤5字，一句话≤20字，金句≤60字 |
| 7 | SUCCESs 6 维度 | 简单/意外/具体/可信/情感/故事 |
| 8 | 视觉清晰 | 手机屏上不糊不挤 |
| 9 | 梗感测试 | 看到后想@朋友 or 自嘲转发 |
