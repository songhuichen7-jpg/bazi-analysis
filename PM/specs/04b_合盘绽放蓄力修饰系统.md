## 一、扩展逻辑

### 1.1 从55到210

原系统：10天干类型 → C(10,2)+10 = **55种唯一组合**
新系统：20类型（10天干×绽放/蓄力）→ C(20,2)+20 = **210种唯一组合**

### 1.2 核心原则：骨架不变，动态微调

- **关系类型**由天干五行决定（天作/滋养/火花/镜像/同频/互补），绽放/蓄力不改变关系类型
- **关系动态**由双方绽放/蓄力状态调整——谁更主导、能量碰撞强度、相处节奏
- 55个基础组合的标签、子标签、角色描述保持不变
- 新增**动态修饰句**（1-2句），附加在基础描述之后

### 1.3 变体数学

| 组合类型 | 基础数 | 每组变体数 | 小计 |
|---------|--------|-----------|------|
| 同天干（镜像） | 10 | 3（双绽/混合/双蓄） | 30 |
| 异天干 | 45 | 4（双绽/A绽B蓄/A蓄B绽/双蓄） | 180 |
| **合计** | **55** | — | **210** |

注：同天干的「A绽B蓄」与「A蓄B绽」对称等价，合并为1种"混合"。

## 二、四种动态状态

### 2.1 双绽放（⚡⚡）

**双方能量都在释放期。**

- **关系基调**：外放型——碰撞更激烈、互动更频繁、火花更多
- **优势**：行动力强，化学反应快，不需要预热
- **张力**：容易"太热"，需要给彼此空间散热

**修饰句模板**：
> 「你们都在释放期，化学反应快得不需要催化剂。唯一的提醒：偶尔给对方留点空间散散热。」

**适用于所有6种关系大类的调整方向**：

| 关系大类 | 双绽放修饰方向 |
|---------|--------------|
| 天作搭子 | 默契外显化，"天然吸引"变成"一拍即合" |
| 滋养搭子 | 能量传递加速，"安静给能量"变成"主动输送" |
| 火花搭子 | 碰撞升级，"有火花"变成"火力全开" |
| 镜像搭子 | 共鸣放大，"照镜子"变成"双份能量" |
| 同频搭子 | 同频外放，"阴阳一体两面"变成"协同作战" |
| 互补搭子 | 各展所长，"各有擅长"变成"各自发光" |

### 2.2 绽放×蓄力（⚡🔋）/ 蓄力×绽放（🔋⚡）

**一方在释放期，一方在积蓄期。**

- **关系基调**：主导型——绽放方更主动、蓄力方更接收
- **优势**：自然分工，不争不抢，节奏互补
- **张力**：绽放方可能"嫌慢"，蓄力方可能"压力大"

**修饰句模板**（A绽放B蓄力）：
> 「A 正在全力释放，B 正在默默积蓄。这个组合里，A 的行动力给 B 安全感，B 的沉稳给 A 定心丸。」

**修饰句模板**（A蓄力B绽放）：
> 「A 正在积蓄能量，B 在全力输出。B 负责开路，A 负责在后方稳住。你们的节奏不一样，但方向是一致的。」

**适用于所有6种关系大类的调整方向**：

| 关系大类 | 绽放×蓄力修饰方向 |
|---------|-----------------|
| 天作搭子 | 默契有主次，"天然吸引"加上"一个带一个稳" |
| 滋养搭子 | 能量方向更明确，给出明确的"谁滋养谁" |
| 火花搭子 | 碰撞减缓，"正面对冲"变成"一攻一守" |
| 镜像搭子 | 同类但状态不同，"照镜子"变成"看到不同阶段的自己" |
| 同频搭子 | 互补更明显，绽放方推进、蓄力方沉淀 |
| 互补搭子 | 分工更清晰，绽放方主攻、蓄力方辅助 |

### 2.3 双蓄力（🔋🔋）

**双方都在积蓄期。**

- **关系基调**：内敛型——相处安静、节奏缓慢、连接深层
- **优势**：不急不抢，都在积累，沉淀感强
- **张力**：容易"太闷"，需要偶尔制造一点意外

**修饰句模板**：
> 「你们都在蓄力期，不急着表现，不赶着推进。这段关系像文火慢炖——外面看不出动静，里面已经入味了。」

**适用于所有6种关系大类的调整方向**：

| 关系大类 | 双蓄力修饰方向 |
|---------|--------------|
| 天作搭子 | 默契内化，"天然吸引"变成"无声默契" |
| 滋养搭子 | 能量缓释，"给能量"变成"陪着就是最好的能量" |
| 火花搭子 | 碰撞降级，"火花"变成"暗流"——表面平静底下有戏 |
| 镜像搭子 | 共鸣深层化，"照镜子"变成"看到最真的自己" |
| 同频搭子 | 同频沉淀，"互补"变成"一起慢慢变好" |
| 互补搭子 | 各自充电，"各有擅长"变成"各自积累，等时机协同" |

### 3.7 实现约束：方向绑定

**滋养搭子**的修饰句区分「给方」和「收方」：
- 给方 = 五行相生链中生出能量的一方（如木生火中，木方为给方）
- 收方 = 接收能量的一方
- 前端需按 `04_合盘系统.md` 中的五行相生表确定A/B谁是给方

**火花搭子**的修饰句区分「克方」和「被克方」：
- 克方 = 五行相克链中发起克制的一方（如木克土中，木方为克方）
- 被克方 = 被克制的一方
- 前端需按 `04_合盘系统.md` 中的五行相克表确定A/B谁是克方

**修饰句索引键** = `(关系大类, 动态状态, [可选]方向枚举)`

### 3.8 去重规则

基础文案（04a）与修饰句可能出现语义重叠。前端组装时遵循：
1. 若基础描述已包含"快/慢""冲/稳"等节奏描述，修饰句可缩短为最后一句（张力提示）
2. 修饰句永远放在基础描述之后，用换行分隔
3. 若产品层决定UI空间不够，修饰句可折叠到"查看更多"

## 五、实现优先级

### 第一优先级：修饰句库覆盖

已在第三节定义了6大关系类型的修饰句库，合计 **23条通用修饰句**：
- 天作搭子：4条（双绽/绽蓄/蓄绽/双蓄）
- 镜像搭子：3条（双绽/混合/双蓄，因同天干对称合并）
- 同频搭子：4条
- 滋养搭子：4条
- 火花搭子：4条
- 互补搭子：4条

这23条修饰句直接覆盖全部210种组合：
- 根据天干关系确定关系大类（6选1）
- 根据双方绽放/蓄力状态确定动态
- 组装：基础文案 + 对应修饰句

### 第二优先级（可选）：个性化修饰

对于5种天干合组合，可额外撰写个性化修饰句（已在3.1节完成）。
其他高流量组合可逐步补充个性化修饰。
