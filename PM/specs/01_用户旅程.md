# 用户旅程设计

> 文档版本：v3.0 | 2026-04-18
> 范围：完整用户流程（裂变层 + 核心产品层）
> 依赖：00_产品架构重构 + 02_八字人格类型系统 + 02a_格局标签系统

---

## 旅程总览

产品有两层，用户旅程跨越两层：

```
裂变层（获客）：
朋友圈看到卡片 → 点链接 → 输入生日 → 3 秒出类型卡片 → 分享/合盘

                          ↓ "想了解更多"

核心产品层（留存 + 价值）：
下滑/点击 → 专栏报告（性格/事业/感情/流年） → 自由对话
```

三条路径：

| 路径 | 触发 | 目标 |
|------|------|------|
| A：个人图鉴 | 看到分享卡片 / 直接访问 | 出类型 → 分享 → 深度解读 |
| B：合盘图鉴 | 收到合盘邀请 | 双人合盘 → 分享 → 相处指南 |
| C：核心探索 | 从 A/B 的结果页深入 | 专栏报告 → 自由对话 |

## 路径 A：个人图鉴

### Stage 1：触达

**场景**：用户在微信朋友圈/群聊/小红书看到一张分享卡片

**卡片要做到**：
1. 信息流中视觉跳出（配色、插画区别于常见内容）
2. 3 秒传达"这是什么" -- 类型名 + 一句话
3. 激发好奇 -- "我是什么底色？"

**入口**：
- 微信 JS-SDK 分享链接（标题 + 描述 + 缩略图）
- 卡片长图带二维码/短链
- 小红书帖子底部链接

### Stage 2：落地页

```
┌────────────────────────────────┐
│    有时 · 人格图鉴             │
│                                │
│    看看你的命盘底色是什么          │
│                                │
│  ┌───────────────────────┐     │
│  │  请选择你的出生日期      │     │
│  │  [年] [月] [日] [时辰]  │     │
│  └───────────────────────┘     │
│                                │
│  [  查 看 我 的 底 色  ]         │
│                                │
│  * 仅用于排盘，不会存储          │
│    （ 已有 XX,XXX 人查看 ）       │
└────────────────────────────────┘
```

**关键设计决策**：

| 决策 | 方案 | 理由 |
|------|------|------|
| 输入项 | 年 + 月 + 日 + 时辰 | 排盘最少需要四项。时辰可选"不确定" |
| 时辰不确定 | 默认"不知道" → 仅用日柱排盘 | 降低门槛，牺牲精度保留核心体验 |
| 性别 | **不收集** | 减步骤。20 型不区分性别 |
| 隐私 | 底部小字"仅用于排盘" | 降顾虑。无注册/登录 |
| 阳历/农历 | 默认阳历，提供切换按钮 | 年轻人多记阳历生日 |

**操作细节**：
- 年份下拉默认定位 1995-2005（目标用户 20-30 岁）
- 月/日联动日历选择器
- 时辰 12 时辰 + "不确定"，附时辰对照说明
- 农历闰月用 Lunisolar.js 处理

### Stage 3：排盘

**等待动画**（1-3 秒）：

```
正在翻阅古籍，解读你的命盘...
[进度动画：翻书/八卦旋转]
```

**后台逻辑**：
1. 公历 → 农历 → 排出四柱（年柱、月柱、日柱、时柱）
2. 日主天干 + 身强/身弱 → 映射 20 型之一（10天干 × 绽放/蓄力）；日干+月支 → 格局标签（10十神 × 绽放/蓄力）
3. 检索典籍断词
4. 组装结果数据 + 预加载专栏报告

## 旅程中的关键指标

### 北极星指标

```
裂变系数 K = 每个用户带来的新用户数
目标：K > 1（自传播增长）
```

### 分层指标

| 阶段 | 指标 | 定义 | 目标 |
|------|------|------|------|
| 触达 → 落地 | **CTR** | 看到卡片后点链接 | > 15% |
| 落地 → 输入 | **输入完成率** | 到达页面后完成输入 | > 60% |
| 结果 → 分享 | **分享率** | 看到结果后分享卡片 | > 30% |
| 结果 → 合盘 | **合盘发起率** | 看到结果后发起合盘 | > 15% |
| 合盘邀请 → 完成 | **合盘完成率** | 收到邀请后完成 | > 40% |
| 结果 → 深入 | **报告查看率** | 看到结果后下滑看报告 | > 40% |
| 报告 → 对话 | **对话转化率** | 看完报告后进入对话 | > 15% |
| 总体 | **K** | 每用户带来新用户数 | > 1.0 |

### 裂变系数计算

```
K = 分享率 × CTR × 输入完成率
乐观：0.35 × 0.20 × 0.65 = 0.046（每次分享）
平均触达 50 人 → K = 0.046 × 50 = 2.3

合盘额外贡献：0.15 × 0.40 = 0.06/用户
合盘用户自带高分享率 → 额外 ~0.5

综合 K ≈ 2.3 + 0.5 = 2.8（极乐观）
保守估计 K ≈ 1.2-1.5
```

## 边界处理

### 时辰不确定

| 情况 | 处理 |
|------|------|
| 选择"不确定" | 仅用年月日三柱。类型映射依然可用（日柱是核心），标注"知道时辰结果更精准" |
| 后续补充时辰 | 提供入口补填，重新排盘。类型不会变（日主不变），但专栏报告内容更精准 |

### 阳历/农历

| 情况 | 处理 |
|------|------|
| 不确定阳历还是农历 | 默认阳历。提供切换按钮 |
| 农历闰月 | Lunisolar.js 处理 |

### 极端日期

| 情况 | 处理 |
|------|------|
| 1900 年之前 | 不支持 |
| 2025 年之后出生 | 不支持 |
| 2 月 29 日 | 正确处理闰年 |

### 合盘邀请

| 情况 | 处理 |
|------|------|
| 链接被第三方点开 | 不限身份，任何人可合盘 |
| 同一邀请多人使用 | 允许。每次独立结果 |
| 邀请方看结果 | 合盘完成后生成结果页 URL，双方可访问 |

### 重复测试

| 情况 | 处理 |
|------|------|
| 输入不同日期 | 允许。每次新结果 |
| 修改上次结果 | 无用户系统，直接重新输入 |

### 微信生态

| 情况 | 处理 |
|------|------|
| 诱导分享 | **绝对不做**"分享后解锁" |
| 外链限制 | (1) 引导浏览器打开 (2) 考虑小程序版本 |
| JS-SDK 限制 | 需企业认证公众号 |

## 旅程优化优先级

| 优先级 | 环节 | 优化方向 |
|--------|------|---------|
| P0 | 输入 → 结果 | 速度 < 3 秒、准确性 |
| P0 | 结果 → 分享 | 卡片质量、分享路径顺滑 |
| P0 | 结果 → 报告 | 下滑引导、报告内容质量 |
| P0 | 分享 → 触达 | 卡片视觉吸引力 |
| P1 | 结果 → 合盘 | 合盘入口引导 |
| P1 | 报告 → 对话 | 对话入口引导、建议问题质量 |
| P1 | 合盘 → 相处指南 | 合盘后深入引导 |
| P2 | 对话质量 | RAG 准确性、回答风格 |
