## 一、合盘的产品定义

**"合盘"不是传统命理的"合婚"。**

传统合婚判断"合不合""该不该结婚"。我们的合盘：
- 看两个人的**能量关系**（互补/同频/碰撞/滋养/默契）
- 给一个**有趣的关系标签**
- 激发分享欲
- **引导进入核心产品**（"你和 TA 的相处指南"专栏报告）

**产品表达**："查八字合盘图鉴 -- 看看你们是什么搭子"

---

## 二、合盘规则

### 规则 1：双方主动

| 场景 | 允许？ | 理由 |
|------|--------|------|
| A 发邀请，B 自己输入生日 | ✅ | 双方主动 |
| A 输入自己和 B 的生日 | ❌ | B 无知情权 |
| A 邀请 B，C 点开邀请 | ✅ | C 主动点开 |

### 规则 2：不做匹配度评分

| 做 | 不做 |
|----|------|
| "你们是点火搭子" | "你们的匹配度是 78%" |
| "一方给另一方能量" | "你们有 3 个不合的地方" |
| 定性的关系标签 | 定量的分数排名 |

### 规则 3：所有结果正面

"不合"的情况用正面重构：

| 传统说法 | 我们的说法 |
|---------|----------|
| "你们不合" | "你们是碰撞型搭子" |
| "你们相克" | "有摩擦才有火花" |
| "性格冲突" | "正因为不同才有化学反应" |

## 三、命理规则

### 3.1 映射逻辑

基础关系矩阵：10天干 × 10天干 = 55种唯一组合（关系类型由天干五行决定）。叠加绽放/蓄力状态后扩展为 **210种** 唯一组合。详见 `04b_合盘绽放蓄力修饰系统.md`。

```
A 的天干 + B 的天干 → 查五行关系 → 确定关系类型 → 查文案矩阵
```

### 3.2 五行关系

| 关系 | 五行逻辑 | 关系基调 |
|------|---------|---------|
| **相生** | A 生 B 或 B 生 A | 滋养型 -- 一方给另一方能量 |
| **相合** | 天干合（5 对） | 默契型 -- 天然吸引，最特别 |
| **相同** | 同五行同阴阳 | 镜像型 -- 照镜子般的共鸣 |
| **同行异相** | 同五行不同阴阳 | 同频型 -- 同频但互补 |
| **相克** | A 克 B 或 B 克 A | 火花型 -- 碰撞出新东西 |
| **无直接关系** | 不生不克不合 | 互补型 -- 各有所长 |

### 3.3 天干合（最特殊的关系）

| 合 | 天干 | 化气 | 含义 | 类型组合 |
|----|------|------|------|---------|
| 甲己合 | 木+土 | 化土 | 中正之合 | 01反脆弱 × 06慢炖 |
| 乙庚合 | 木+金 | 化金 | 仁义之合 | 02柔韧 × 07硬核 |
| 丙辛合 | 火+金 | 化水 | 威制之合 | 03自燃 × 08精装 |
| 丁壬合 | 火+水 | 化木 | 淫慝之合 | 04闪念 × 09改道 |
| 戊癸合 | 土+水 | 化火 | 无情之合 | 05定海神针 × 10渗透 |

天干合是最有戏剧性的关系 -- 给出最独特的标签和描述。

### 3.4 五行相生

| 相生 | 方向 | 含义 |
|------|------|------|
| 木生火 | 01/02 → 03/04 | 给你能量，让你发光 |
| 火生土 | 03/04 → 05/06 | 热情温暖了你的沉稳 |
| 土生金 | 05/06 → 07/08 | 沃土里长出锋芒 |
| 金生水 | 07/08 → 09/10 | 精雕细琢后汇入大海 |
| 水生木 | 09/10 → 01/02 | 滋润让你扎根更深 |

### 3.5 五行相克（正面重构）

| 相克 | 方向 | 正面重构 |
|------|------|---------|
| 木克土 | 01/02 ↔ 05/06 | "你给 TA 松了土" -- 不是破坏，是疏通 |
| 土克水 | 05/06 ↔ 09/10 | "你给 TA 定了方向" -- 不是拦住，是引导 |
| 水克火 | 09/10 ↔ 03/04 | "你让 TA 冷静下来" -- 不是浇灭，是降温 |
| 火克金 | 03/04 ↔ 07/08 | "你把 TA 锻造成器" -- 不是伤害，是磨砺 |
| 金克木 | 07/08 ↔ 01/02 | "你给 TA 修了枝" -- 不是砍伐，是塑形 |

## 四、10×10 合盘关系矩阵

> 注：绽放/蓄力状态不改变关系类型，只影响关系动态。55种基础关系叠加4种动态状态（双绽放/绽蓄/蓄绽/双蓄力）后产生210种变体，详见04b。

### 关系大类

所有 55 种组合归入 6 大关系类型：

| 关系类型 | 五行基础 | 调性 |
|---------|---------|------|
| 🤝 **天作搭子** | 天干合 | 天然默契，最特别（5 种组合） |
| 🌱 **滋养搭子** | 相生 | 一方给另一方能量（20 种组合） |
| 🔥 **火花搭子** | 相克 | 碰撞出新东西（20 种组合） |

| 🪞 **镜像搭子** | 同天干 | 照镜子般共鸣（10 种组合） |
| 🔄 **同频搭子** | 同五行异阴阳 | 同频但有差异（5 种组合） |

### 矩阵逻辑说明

**天干合（🤝）优先级最高**：如果两个天干构成天干合，忽略其五行生克关系，直接归为"天作搭子"。

**相生方向**：
- 木(甲乙) 生 火(丙丁)
- 火(丙丁) 生 土(戊己)
- 土(戊己) 生 金(庚辛)
- 金(庚辛) 生 水(壬癸)
- 水(壬癸) 生 木(甲乙)

**相克方向**：
- 木(甲乙) 克 土(戊己) -- 但甲己合除外
- 土(戊己) 克 水(壬癸) -- 但戊癸合除外
- 水(壬癸) 克 火(丙丁) -- 但丁壬合除外
- 火(丙丁) 克 金(庚辛) -- 但丙辛合除外
- 金(庚辛) 克 木(甲乙) -- 但乙庚合除外

## 五、各关系类型的标签风格

### 🤝 天作搭子（天干合 -- 5 种）

最特别、最有戏剧性。强调"天然默契""命盘里自带的缘分"。

| 组合 | 合 | 标签方向 | 示例 |
|------|---|---------|------|
| 甲01 × 己06 | 甲己合 | 刚柔并济 | "你扛住，我撑着" |
| 乙02 × 庚07 | 乙庚合 | 柔能化刚 | "你砍不动我，但我绕得进你" |
| 丙03 × 辛08 | 丙辛合 | 热情遇冷静 | "你发光，我折射" |
| 丁04 × 壬09 | 丁壬合 | 灵感碰自由 | "你一闪，我就接住" |
| 戊05 × 癸10 | 戊癸合 | 厚重遇灵动 | "你定海，我渗透" |

### 🌱 滋养搭子（相生 -- 20 种）

一方给另一方能量的温暖关系。

标签风格：`形容词 + 关系` -- "温暖后盾""能量补给站""光合搭档"

### 🔥 火花搭子（相克 -- 20 种）

有碰撞有火花的关系。**全部正面重构**。

标签风格：`动词 + 名词` -- "碰撞火花""互相磨刀""点燃彼此"

### 🧩 互补搭子（无直接关系）

各有所长的搭配。

标签风格：`对比结构` -- "你攻我守""一快一稳""破局搭子"

### 🪞 镜像搭子（同天干 -- 10 种）

同类型相遇的共鸣。

标签风格：`重复 + 共鸣` -- "同频共振""懂你的人""另一个你"

### 🔄 同频搭子（同五行异阴阳 -- 5 种）

| 组合 | 关系 | 标签方向 |
|------|------|---------|
| 甲01 × 乙02 | 阳木+阴木 | 大树和花草 |
| 丙03 × 丁04 | 阳火+阴火 | 太阳和烛火 |
| 戊05 × 己06 | 阳土+阴土 | 城墙和田园 |
| 庚07 × 辛08 | 阳金+阴金 | 刀剑和珠玉 |
| 壬09 × 癸10 | 阳水+阴水 | 江河和雨露 |

## 六、合盘文案结构

每个合盘需要以下元素（由 AI 生成 → 人工筛选）：

```
1. 关系标签（≤4字）
2. 三个子标签（各≤8字）
3. A 的角色描述（≤8字）
4. B 的角色描述（≤8字）
5. 关系描述（2句话）
6. CTA 金句（1句，可转发给对方）
```

文案生成的 few-shot prompt 详见 `specs/07_标签文案生成系统.md`。

### 文案示例（每种关系类型各一个）

**🤝 天作搭子 · 丁04闪念 × 壬09改道（丁壬合）**

- 标签：天生搭子
- 子标签：天干合局 / 灵感碰自由 / 想到一起去
- A 角色：灵光一闪
- B 角色：说走就走
- 描述："你的灵感一来，TA 立刻就能接住。你们的默契不是练出来的，是命盘里自带的。"
- CTA："你一说我就懂。"

**🌱 滋养搭子 · 甲01反脆弱 × 丁04闪念（木生火）**

- 标签：点火搭子
- 子标签：你是燃料我是火 / 安静地给对方能量 / 一点就着
- A 角色：你的存在就是能量
- B 角色：接到就能亮
- 描述："你不需要做什么，你在那里，TA 就能找到灵感。你们之间的能量传递，安静但有效。"
- CTA："有你在我就有电。"

**🔥 火花搭子 · 戊05定海神针 × 壬09改道（土克水 → 引导型）**

- 标签：河堤搭子
- 子标签：你定方向我冲锋 / 有边界才有力量 / 不是拦你是导你
- A 角色：立在那里不动摇
- B 角色：绕不过就冲过去
- 描述："一个是拦不住的水，一个是搬不动的墙。听起来要打架？不，是水终于有了方向。"
- CTA："别拦我。好吧，你拦了我反而知道往哪冲了。"

**🪞 镜像搭子 · 丙03自燃 × 丙03自燃**

- 标签：双日搭子
- 子标签：两个太阳 / 走到哪亮到哪 / 快乐加倍
- A 角色：发光发热
- B 角色：一样发光发热
- 描述："两个太阳凑在一起，不是互相抢光，是照亮范围直接翻倍。"
- CTA："咱俩也太亮了。"

**🔄 同频搭子 · 庚07硬核 × 辛08精装（阳金+阴金）**

- 标签：刚柔搭子
- 子标签：一个砍一个磨 / 粗细搭配 / 互相成就
- A 角色：上手就干
- B 角色：精雕细琢
- 描述："你负责开路，TA 负责打磨。你们做出来的东西，又快又好。"
- CTA："你开路，我收尾。"

## 七、绝对不出现的内容

| 红线 | 替代 |
|------|------|
| "你们不合" | "你们是碰撞型搭子" |
| "不适合在一起" | 不做关系建议 |
| "你们会吵架" | "你们在一起永远不会无聊" |
| 匹配度打分 | 只有定性标签 |
| 建议分手/不交往 | 无关系建议 |
| "相冲""相刑""相害" | 不用命理术语 |

## 八、技术流程

```
A 点击"邀请好友合盘"
    ↓
A 输入昵称
    ↓
系统生成邀请链接：
chabazi.com/hepan?inviter_type=01&inviter_name=小满&inviter_dm=甲
    ↓
A 发给 B
    ↓
B 打开链接 → "@小满 邀请你来合盘"
    ↓
B 输入昵称 + 生日
    ↓
排盘 → 确定 B 类型
    ↓
查 10×10 矩阵 → 获取关系标签和文案
    ↓
展示合盘结果 → 生成合盘卡片
    ↓
引导进入"你和 TA 的相处指南"（核心产品层）
```

### 数据存储

纯前端方案：

| 数据 | 存储？ | 方式 |
|------|--------|------|
| 用户生日 | **不存储** | 仅前端计算 |
| 排盘结果 | 编码在 URL | Base64 |
| 合盘结果 | 确定性计算 | 无需存储 |
| 昵称 | URL 参数 | — |

**零后端存储，零隐私风险。**

## 九、补充考虑

20型人格与绽放/蓄力动态如何与合盘矩阵衔接、以及 **210种** 组合扩展的完整定义，见 `04b_合盘绽放蓄力修饰系统.md`。

### 同性合盘

完全支持。"搭子"不限恋爱关系。朋友、同事、室友都可以。文案避免"情侣""恋人"等限定词。

### 多人合盘

暂不支持。每次两人。

### 二次合盘

A 可以和 B、C、D 分别合盘，不限次数。本身就是裂变路径。

### 合盘 → 核心产品衔接

合盘结果页底部增加**"你和 TA 的相处指南"**入口 -- 这是第五份专栏报告，覆盖：
- 相处中谁主动谁跟随
- 可能的摩擦点和化解方式
- 最佳协作方式

这个衔接让合盘不只是"看一眼标签就走"，而是引导用户进入核心 AI 产品层。
