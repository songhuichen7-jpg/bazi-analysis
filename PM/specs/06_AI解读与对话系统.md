# AI 解读与对话系统设计

> 文档版本：v3.0 | 2026-04-18
> 本文档定义核心产品的两种模式：专栏报告（预生成）+ 自由对话（实时 LLM）
> **类型层**：20 种基础类型（10 天干 × 绽放/蓄力）+ 20 种格局标签（10 十神 × 绽放/蓄力，月令动态层）；专栏性格叙事与对话须两层同参，格局标签（十神）详见 `02a_格局标签系统.md`。

---

## 一、两种模式的定位

| 模式 | 覆盖需求 | 生成方式 | Token 成本 | 用户比例 |
|------|---------|---------|-----------|---------|
| **专栏报告** | 高频刚需（性格/事业/感情/流年） | 预生成模板 + LLM 一次性生成 | 低（可缓存） | ~60% 用户会看 |
| **自由对话** | 长尾需求（任何问题） | 实时 LLM 多轮对话 | 高（每轮消耗） | ~20% 用户会用 |

## 三、自由对话系统

### 对话架构

```
┌────────────────────────────────┐
│  [专栏]  [对话]                 │  ← 切换入口
│                                │
│  你好！我已经看过你的命盘了。     │  ← AI 开场
│  你可以问我任何关于命盘的问题。   │
│                                │
│  ┌──────────┐ ┌──────────┐     │
│  │ 我的正缘  │ │ 今年运势  │     │  ← 建议问题（动态生成）
│  │ 画像？    │ │ 怎么样？  │     │
│  └──────────┘ └──────────┘     │
│  ┌────────────────────────┐    │
│  │ 最近为什么总遇到烂桃花？ │    │
│  └────────────────────────┘    │
│                                │
│  ┌──────────────────────────┐  │
│  │ 你想知道什么？             │  │  ← 输入框
│  │                      [发送]│  │
│  └──────────────────────────┘  │
│                                │
└────────────────────────────────┘
```

### System Prompt 设计

```
你是"查八字"的 AI 命理分析师。你精通子平八字命理，熟读以下典籍：
- 《三命通会》（万民英·明）
- 《渊海子平》（徐大升·宋）
- 《滴天髓阐微》（任铁樵·清）
- 《穷通宝鉴》

当前用户的命盘信息：
- 四柱：{年柱} {月柱} {日柱} {时柱}
- 日主：{天干}（{五行}{阴阳}）
- 子平格局（月令）：{格局名称}
- **格局标签（产品层，十神×绽放/蓄力）**：{格局标签完整名，如「食神格·绽放版：天生享乐家」}
- **命盘底色类型（天干×绽放/蓄力）**：{类型名，如「甲·绽放·参天木命」}
- 格局标签金句（供语气对齐，勿照抄堆砌）：「{格局标签金句}」
- 当前大运：{大运天干地支}（{起始年}-{结束年}）
- 当前流年：{流年天干地支}（{年份}）

{RAG 检索到的相关典籍段落}

## 回答规范

1. **引用典籍**：每个关键判断至少引用一处典籍原文，格式为"《书名》云：'原文'"
2. **语言风格**：
   - 用"你"直接称呼用户
   - 每句不超过 25 字
   - 不使用"日主""偏财""正官""食神格"等术语；用户可见的底色名、格局标签产品名可在必要时自然出现，但解释仍用白话
   - 像一个懂命理的朋友在聊天，不像算命先生在卖弄
3. **态度**：
   - 不做宿命论断言。用"倾向""容易""适合"等词
   - 不回避不利信息，但用正面重构表达
   - 给出可操作的建议，不只是描述
4. **边界**：
   - 不预测具体事件发生日期
   - 不做健康诊断
   - 不替用户做人生决策
   - 涉及"不太好"的方面时，框架为"需要注意"而非"你会如何"
```

### 建议问题的动态生成逻辑

建议问题不是写死的，而是根据用户命盘动态生成：

```
IF 用户当前大运走财星：
  建议问题 += "我这步运适合投资吗？"
  建议问题 += "最近花钱大手大脚，正常吗？"

IF 用户当前流年有桃花：
  建议问题 += "今年感情方面有什么变化？"
  建议问题 += "我的正缘什么时候出现？"

IF 用户日支有冲/刑：
  建议问题 += "我和另一半的相处模式是什么？"

IF 用户即将换大运：
  建议问题 += "我马上要换大运了，有什么变化？"

ALWAYS:
  建议问题 += "我适合什么工作方向？"
  建议问题 += "我今年需要注意什么？"
```

### 对话的 RAG 检索策略

用户提问 → 语义分类 → 检索对应典籍段落 → 注入 prompt

| 问题类别 | 检索来源 | 优先典籍 |
|---------|---------|---------|
| 性格相关 | 干支体象 + 论性情 + 十干坐支 | 渊海子平、三命通会·卷七 |
| 事业相关 | 格局论 + 大运分析 | 三命通会·卷二-六、穷通宝鉴 |
| 感情相关 | 六亲论·夫妻 + 日支分析 | 滴天髓·夫妻、渊海子平·六亲论 |
| 流年相关 | 流年太岁 + 大运交互 | 三命通会·卷七-九 |
| 健康相关 | 论疾病 | 三命通会·卷七、渊海子平·论疾病 |
| 合盘相关 | 天干合 + 五行生克 + 六亲 | 三命通会·卷二、滴天髓·夫妻 |

## 四、生成成本控制

### 专栏报告

每份报告约 800-1200 字，按 MiMo V2 Flash 计价：

| 项目 | 值 |
|------|---|
| 输入 Token（prompt + 典籍段落） | ~3000 tokens |
| 输出 Token（报告正文） | ~1500 tokens |
| 单次成本 | ~¥0.01 |
| 5 份报告 | ~¥0.05/用户 |
| 10000 DAU | ~¥500/天 |

**优化策略**：
- 相同日柱的用户（60 种日柱 × 12 月令 = 720 种组合），报告可以缓存复用
- 只有流年报告需要年度更新
- 缓存命中后成本趋近于零

### 自由对话

| 项目 | 值 |
|------|---|
| 单轮输入 Token | ~2000 tokens（prompt + context） |
| 单轮输出 Token | ~500 tokens |
| 平均对话轮数 | 5-10 轮 |
| 单次对话成本 | ~¥0.05-0.10 |
| 日活 2000 对话用户 | ~¥100-200/天 |

## 五、报告与对话的衔接

报告不是终点，而是对话的引子：

```
[专栏报告·事业蓝图]

... 你的事业方向适合 XX 领域 ...

想了解更多？可以问我：
┌────────────────────────┐
│ "我适合考公还是进大厂？" │
│ "明年跳槽合适吗？"       │
└────────────────────────┘
    ↓ 点击后进入对话模式
```

每份报告底部都有"想深入聊聊？"的入口，引导用户进入对话。对话中 AI 可以引用报告中已分析的内容，保持一致性。
