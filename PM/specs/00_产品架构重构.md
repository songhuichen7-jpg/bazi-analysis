# 产品架构重构

> 文档版本：v3.0 | 2026-04-18
> 本文档是对 v1 全部 spec 的修正，重新定义产品的两层架构和核心功能

---

## 一、v1 犯的错

把分享卡片当成了全部产品。实际上：
- **卡片/合盘是裂变层** -- 获客入口，让人看一眼就想测
- **AI 解读 + AI 对话才是核心产品** -- 让人觉得"这东西真的有用"
- 没有核心产品，用户看完卡片就走了 -- 没有留存，没有付费场景

## 三、用户旅程（修正版）

### 路径 A：从分享卡片进入（主要路径）

```
看到朋友的分享卡片
  ↓
点击链接 → 落地页
  ↓
输入生日（年月日 + 时辰可选）
  ↓
3 秒出结果 → 类型卡片
  ↓
用户在这里分叉：
  ├── 60% 保存/分享卡片 → 裂变
  ├── 15% 发起合盘邀请 → 社交裂变
  └── 40%+ "想看完整解读" → 进入核心产品
```

### 路径 B：进入核心产品

```
点击"看完整解读" / 从卡片页向下滚动
  ↓
进入主界面（双面板布局）：
┌──────────────────┬───────────────────┐
│   命盘面板        │    对话面板        │
│                  │                   │
│  四柱排盘         │  "你想了解什么？"  │
│  大运走势         │                   │
│  流年标记         │  [性格] [事业]     │
│                  │  [感情] [流年]     │
│  专栏报告导航      │                   │
│  • 性格全景       │  建议问题按钮：     │
│  • 事业运         │  "我的正缘画像？"  │
│  • 感情运         │  "今年适合创业吗？" │
│  • 2026流年       │  "最近运气为什么差" │
│                  │                   │
└──────────────────┴───────────────────┘
```

### 核心产品的使用模式

**模式 1：专栏报告（高频刚需）**

AI 预生成结构化报告，用户直接阅读。覆盖大多数用户最想知道的几个领域。

参考你自己的算命经历，高频需求排序：

| 需求 | 频率 | 报告化方式 |
|------|------|----------|
| **性格全景** | 100%（所有人） | "你的命格故事" -- 性格、优势、盲区 |
| **事业运** | ~70% | "你的事业蓝图" -- 适合什么方向、何时发力 |
| **感情运** | ~60% | "你的感情图谱" -- 正缘画像、感情时间线 |
| **年度运势** | ~50% | "你的 2026" -- 关键节点、注意事项 |

**模式 2：自由对话（长尾需求）**

用户可以自由提问，AI 基于排盘结果 + 典籍知识回答。

长尾需求举例：
- "我的正缘长什么样？"
- "我和 XX 生日的人合不合？"
- "为什么我最近总遇到烂桃花？"
- "秋招运势怎么样？"
- "我的健康要注意什么？"
- "我适合考研还是工作？"

## 四、类型系统重新思考

### v1 的问题

- 固定 8 种太武断
- 文案太安全，没有"梗感"
- 把标签写死了，应该用 AI 基于典籍生成

### 分析：多少种类型合适？

| 数量 | 参考 | 优势 | 劣势 |
|------|------|------|------|
| 5（五行） | — | 极简，视觉辨识度高 | 太粗，每种 20%，差异化不足 |
| 10（十天干） | — | 1:1 映射日主，古籍直接支持 | 略多，但可接受 |
| 12（十二地支/月份） | 星座 | 大众熟悉的框架 | 与日主脱节 |
| 27（多维组合） | SBTI | 极高差异化 | 太复杂 |

### 建议方案：20 型（十天干 × 绽放/蓄力）+ 格局标签

类型体系拆为**基础层 + 动态层**，详见 `specs/02_八字人格类型系统.md`、`specs/02a_格局标签系统.md`。

**基础层（20 种）** = 10 天干 × 2 状态（**绽放型** / **蓄力型**）

- **绽放型**：日主偏强（得令、得助等综合判定），能量外放、释放期；产品化表述对接「身强」意象。
- **蓄力型**：日主偏弱，能量内收、积蓄期；产品化表述对接「身弱」意象。
- 命名示例：**参天木·绽放型**、**参天木·蓄力型**。

**动态层（20 种格局标签）** = 10 十神 × 2 状态（与绽放/蓄力同一套强弱语境）

- 依据 **日干 + 月支** 按月令取格逻辑确定十神格局，再叠加绽放/蓄力版本。
- 命名示例：**食神格·天生享乐家**、**伤官格·锋芒外放者**。

**卡片可见组合**：20（基础）× 20（格局标签）= **200** 种主展示组合（受天干×月令约束，实际分布有偏，但用户感知上以 200 档差异化为主轴）。

**合盘侧**：双人互动修饰模板已扩展到 **210** 种组合规模，详见 `specs/04b_合盘绽放蓄力修饰系统.md`（与单人 200 档类型体系配套，维度不同）。

### 标签的最终生成方式

不是由我们手写每一条 → 而是：

1. 我定义**风格规范和 few-shot 示例**
2. 基于典籍断词，用 LLM 批量生成候选标签
3. 两人从候选中挑选/修改最有爆点的
4. 预写固定到产品中

## 五、核心 AI 解读系统设计

### 专栏报告结构

每份报告都遵循统一框架：

```
[标题] 你的 XX 分析
[典籍引用] 《三命通会》论你的日柱："原文..."
[白话解读] 用"你"开头的现代语言解读
[关键标签] 3-5 个标签化的要点
[时间线]（如果适用）关键年份节点
[行动建议] 1-2 条可操作的建议（不是"宿命论"）
```

### 高频专栏（预生成 + LLM 增强）

**专栏 1：你的命格故事（性格全景）**
- 来源：天干体象诗 + 五行性情论 + 十干坐支 + 纳音
- 内容：核心性格、天然优势、潜在盲区、在别人眼中的样子
- 格式：叙事体，像在讲一个人的故事

**专栏 2：你的事业蓝图**
- 来源：格局分析 + 大运流年
- 内容：适合什么方向、何时是发力期、当前阶段建议
- 格式：时间线 + 分析

**专栏 3：你的感情图谱**
- 来源：日支夫妻宫 + 财星/官星分析 + 桃花
- 内容：正缘画像、感情模式、关键时间节点
- 格式：画像描述 + 时间线

**专栏 4：你的 2026**
- 来源：流年干支 + 大运交互
- 内容：今年关键词、机会窗口、需要注意的月份
- 格式：月度时间线 + 关键事件提醒

### 自由对话系统

**Prompt 架构**：

```
[System Prompt]
你是一位精通子平八字命理的分析师。
用户的四柱为 {四柱}，日主为 {日主}，格局为 {格局}。
大运：{当前大运}
流年：{当前流年}

你有以下典籍知识可以引用：
{RAG 检索的相关典籍段落}

回答要求：
- 始终引用具体典籍原文作为依据
- 用"你"开头，像和朋友聊天一样
- 每句不超过 25 字
- 不做宿命论断言，用"倾向""容易""适合"等词
- 给出积极的、可行动的建议
- 不回避"不太好"的信息，但用重构方式表达
```

**建议问题按钮**：根据用户的命盘动态生成最相关的问题。

比如如果用户当前大运走偏财，建议问题可能包括：
- "我这步运适合投资吗？"
- "为什么最近花钱大手大脚？"
- "我的财运转折点在什么时候？"

## 七、文案评估结果（传播学专家 + 目标用户）

### 传播学专家核心意见

1. **10 型是甜点区** -- 8 个偏少，10 个刚好。不建议堆更多类型名，而应用"主标签 + 副标签"增加差异化
2. **B 方案的骨架更利于传播**，但需要把"指向他人的攻击"改为"自我叙事"
3. 建议做到**"可转发台词的力度"**，不必达到 SBTI 式的"全网吵架力度"
4. 对外展示 10 个主类型；对内用规则生成"副词/副意象"提高命中，而不增加记忆负担

### 目标用户（23F 应届生）核心反馈

**最想分享 Top 3**（原因：自我安慰型，分享阻力低）
1. 烛灯火命 · 闪念型 -- "最像我日常学习与赶工的心理节奏"
2. 田园土命 · 慢炖型 -- "求职焦虑期最好拿来安慰自己"
3. 珠玉金命 · 精装型 -- "体面、不过分鸡血"

**最不想分享 Bottom 3**（原因：太攻击 or 太鸡汤）
1. 城墙土命 · 钉子户型 -- "梗好笑但怕被误读"
2. 刀剑金命 · 粗加工型 -- "'糙'不想在公开人设里认领"
3. 太阳火命 · 自燃型 -- "略鸡汤中二"

**测完最想继续了解的内容**：
1. 和朋友的相处提示（立刻能聊）
2. 最近求职/面试心态提醒（当下刚需）
3. 适合的工作风格/岗位类型
4. 穿搭/生活方式（加分但非核心）

**"XX命"接受度**：能接受，但需要轻松声明（"娱乐向性格隐喻"）

### 标签修订方向

基于反馈，需要修订的标签（降攻击性，升场景感）：

| 原标签 | 问题 | 修订方向 |
|--------|------|---------|
| "你越压我越硬" | "反脆弱"偏鸡汤 | → "压力来了我不躺" |
| "看着好欺负，其实最难搞" | "最难搞"引战 | → "看着随和，底线焊死" |
| "自己发光不需要别人点" | 太正能量，中二 | → "不等聚光灯，自己开机" |
| "你搬不动我，别试了" | "钉子户"攻击感 | → "认准了就不挪窝" |
| "糙，但砍什么都行" | "糙"自贬 | → "不挑活，上手就干" |
| "拦不住，绕也要绕过去" | "野路子"可能冒犯 | → "正面刚不过，我就改河道" |

## 八、专栏报告的需求论证（数据支撑）

### 后浪研究所 2025 数据（668 份问卷）

| 需求 | 关注度 | 优先级 |
|------|--------|--------|
| **事业运** | 76.5% | #1 |
| **财运** | 74.9% | #2 |
| **爱情运** | 49.6% | #3 |

**关键发现**：事业 > 财运 >> 爱情。7 成年轻人在求事业。

### 结合目标用户反馈的专栏规划

| 专栏 | 覆盖需求 | 优先级 | 形式 |
|------|---------|--------|------|
| **你的命格故事** | 性格认知（100%刚需） | P0 | 叙事体 |
| **你的事业蓝图** | 事业+财运（76%+75%） | P0 | 时间线+分析 |
| **你的感情图谱** | 感情运（50%） | P1 | 正缘画像+时间线 |
| **你的 2026** | 流年运势（高频） | P0 | 月度指南 |
| **你和 TA 的相处指南** | 社交关系（用户最想要的） | P0 | 合盘延伸 |

注意："你和 TA 的相处指南"从用户反馈看优先级最高（"立刻能聊"），但依赖合盘功能，作为合盘的延伸内容。

```
┌────────────────────────────┐

│                            │
│   当前显示对应面板内容       │

```

或者**卡片式下滑**：
```
┌────────────────────────────┐
│  类型结果卡片               │  ← 第一屏（裂变层）
│  [保存] [分享] [合盘]       │
├────────────────────────────┤
│  ↓ 下滑                    │
│  专栏报告（性格/事业/感情）   │  ← 第二屏（核心层入口）
├────────────────────────────┤
│  ↓ 继续下滑                 │
│  对话入口                   │  ← 第三屏（深度层）
│  "想问更多？和你的命盘聊聊"   │
└────────────────────────────┘
```

**推荐方案**：移动端用"卡片式下滑"（上方类型卡片 → 中间专栏报告 → 底部对话入口），桌面端用双面板。原因：
1. 用户主要在手机上使用（微信内打开）
2. 下滑是最自然的探索手势
3. 每一屏都有独立价值（即使不往下看也能分享第一屏）
