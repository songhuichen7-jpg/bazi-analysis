## 二、技术架构概览

### MVP 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React/Next.js | 快速开发 + SSR |
| 部署 | Vercel / Cloudflare Pages | 静态优先 + Edge Functions |
| 排盘 | 前端计算（Lunisolar.js / bazi 库） | 零后端依赖 |
| 卡片生成 | html2canvas | 客户端生成 |
| AI 对话/报告 | MiMo 系列 API | 平衡性能和成本 |
| RAG | 典籍知识库 + 向量检索 | 精准引用 |
| 埋点 | Umami（自部署） | 轻量、隐私友好 |

### 两种 AI 模式的技术差异

| | 专栏报告 | 自由对话 |
|--|---------|---------|
| 生成时机 | 首次访问时一次性生成 | 每轮实时生成 |
| 缓存 | 可缓存（同命盘同报告） | 不可缓存 |
| Token 消耗 | 低（生成一次 ~2000 token） | 高（每轮 ~500 token） |
| Prompt | 结构化模板 | System Prompt + 上下文 |
| RAG | 预定义检索规则 | 动态检索 |

## 三、商业化设计

### 核心原则

```
首要目标是传播（cover 成本就行 → "佛系"）
→ 免费层要足够好，好到用户愿意分享
→ 付费层是额外惊喜，不是缺失的补偿
```

### 免费/付费分层

| 层级 | 内容 | 价格 |
|------|------|------|
| **免费** | 20型底色 + 格局标签结果 + 卡片 + 合盘 + 命格故事报告（P0 报告） | ¥0 |
| **付费 Tier 1** | 完整专栏报告（事业/流年/感情/相处指南） | ¥1.99/份 或 ¥4.99 全部 |
| **付费 Tier 2** | 自由对话（LLM 实时回答） | ¥9.99/10 次对话 或 月卡 ¥19.9 |

### 为什么免费层包含卡片 + 命格故事

1. **卡片是裂变载体** -- 付费卡片 = 裂变断裂
2. **命格故事是 hook** -- 先给价值，才有付费意愿
3. **合盘是社交货币** -- 付费合盘 = 用户不好意思邀请朋友

### 付费墙位置

```
[免费] 输入生日 → 类型卡片 → 分享/合盘 → 命格故事
                                           ↓
                             "想看事业/感情/流年分析？"
                                           ↓
                             [付费] 专栏报告
                                           ↓
                             [付费] 自由对话
```

### 付费转化率预估

| 指标 | 估值 |
|------|------|
| UV → 完成测试 | 60% |
| 完成测试 → 分享 | 30% |
| 完成测试 → 看免费报告 | 40% |
| 免费报告 → 查看付费介绍 | 20% |
| 查看付费介绍 → 付费 | 10-15% |
| **UV → 付费** | **~1.5-2%** |

### 收入/成本测算

**收入**：

| 场景 | 日 UV | 付费率 | ARPU | 日收入 | 月收入 |
|------|-------|--------|------|--------|--------|
| 冷启动 | 1,000 | 1.5% | ¥3 | ¥45 | ¥1,350 |
| 小爆 | 10,000 | 1.5% | ¥4 | ¥600 | ¥18,000 |
| 大爆 | 100,000 | 2% | ¥5 | ¥10,000 | ¥300,000 |

**成本**：

| 项目 | 冷启动月成本 | 大爆月成本 |
|------|------------|-----------|
| 域名 | ¥50 | ¥50 |
| CDN/Hosting | ¥0（免费额度） | ¥200-500 |
| LLM Token（报告 + 对话） | ¥200 | ¥10,000-20,000 |
| RAG 向量库 | ¥0（自部署） | ¥500 |
| **总计** | **~¥250** | **~¥20,000** |

**结论**：冷启动几乎零成本。大爆场景收入远大于成本。"cover 运营和 token 成本"轻松达标。

### LLM Token 成本详估

假设使用 MiMo-V2（~¥0.5/万 token）：

| 场景 | 每日调用量 | 平均 token/次 | 日成本 |
|------|----------|-------------|--------|
| 专栏报告（日 UV 1万） | ~4,000 次 | 2,000 token | ¥4 |
| 自由对话（日 UV 1万） | ~1,500 轮 | 800 token | ¥0.6 |
| **合计** | | | **~¥5/天** |

报告可缓存（同日柱 = 同报告），实际成本更低。60 个日柱 × 5 份报告 = 300 份报告缓存即覆盖所有用户。

### 支付方式

| 方式 | 适用 | 门槛 |
|------|------|------|
| 微信支付（JSAPI/小程序） | 微信内 | 企业资质 |
| 支付宝 H5 | 非微信 | 企业资质 |
| 爱发电/赞赏码 | 极简方案 | 个人即可 |

**建议**：先用赞赏码/爱发电做最简验证。验证付费意愿后再接正式支付。

## 四、冷启动策略

> 两人都是即将毕业的大学生，基本无 KOL 资源。所有策略基于零预算。

### 策略 1：名人/爱豆八字图鉴（内容种子）

用回测过的话题人物做社区传播帖：

| 内容类型 | 示例 | 发布平台 |
|---------|------|---------|
| "XX 是什么底色" | "IU 是珠玉金命·精装型" | 小红书/微博 |
| "XX 和 XX 的合盘" | "XXX 和 YYY 的合盘是天作搭子" | 豆瓣/微博 |
| "20 型人格你是哪个" | 总览图 + 引导测试 | 小红书 |

**要点**：
- 选目标用户喜欢的人物（爱豆、影视角色）
- 不做"算命预测"，做"性格解读"
- 每篇帖子底部附产品链接

### 策略 2：大学校友群/宿舍群种子

直接在身边的社交圈发起：

1. 自己先测 → 发朋友圈（创作者本人）
2. 同学测完 → 合盘 → 发群里
3. 自然裂变

### 策略 3：创作者故事

"两个大学生用 AI 做了一个八字人格图鉴" -- 这本身就是传播故事。

| 平台 | 内容角度 |
|------|---------|
| 即刻/V2EX | 技术角度：vibe coding + AI + 命理 |
| 小红书 | 产品角度：为什么做、怎么做的 |
| Twitter/X | 英文版故事（海外传播） |

## 五、数据指标体系

### 北极星

```
K = 每个用户带来的新用户数
目标：K > 1
```

### 指标分层

#### L1：核心（每天看）

| 指标 | 定义 | 目标 |
|------|------|------|
| **日新增 UV** | 当日独立访客 | 增长趋势 |
| **分享率** | 测完后分享卡片比例 | > 30% |
| **K 值** | 每用户带来新用户数 | > 1.0 |
| **合盘发起率** | 测完后发起合盘比例 | > 15% |
| **报告查看率** | 测完后查看专栏报告比例 | > 40% |

#### L2：转化（每周看）

| 指标 | 定义 | 目标 |
|------|------|------|
| 落地页 → 输入 | 完成输入比例 | > 60% |
| 合盘邀请 → 完成 | 对方完成合盘比例 | > 40% |
| 报告 → 对话 | 看完报告后进入对话比例 | > 15% |
| 付费转化率 | UV → 付费 | > 1.5% |

#### L3：质量（每月看）

| 指标 | 定义 |
|------|------|
| 类型分布 | 20型 + 格局标签组合实际分布是否健康 |
| 各类型分享率 | 哪个类型最有分享欲 |
| 合盘热图 | 哪些组合最受欢迎 |
| 渠道来源 | 微信/小红书/微博/直接 |
| 对话深度 | 平均对话轮数 |
| 次日/七日留存 | 多少人回来 |

## 六、风险与应对

### 合规风险

| 风险 | 等级 | 应对 |
|------|------|------|
| "封建迷信"定性 | 中 | 定位"性格图鉴"，不做命运预测 |
| AI 生成内容监管 | 低 | 报告缓存可审核；对话加内容安全过滤 |
| 个人信息收集 | 低 | 不存储个人数据，前端计算 |
| 微信封链接 | 高 | 备用域名 + 小程序双轨 |

### 运营风险

| 风险 | 等级 | 应对 |
|------|------|------|
| 爆火后挂 | 低 | 静态站 + CDN + 报告缓存 |
| 被抄袭 | 高 | 典籍断词 + AI 对话 + 文案质量是壁垒；先发优势 |
| 热度过后无留存 | 中 | 合盘是天然留存 + 专栏报告给深度价值 + 对话模式给长期使用理由 |
| 负面舆论 | 低 | 不做命运预测，不做"命不好"判断 |
| LLM 幻觉 | 中 | RAG + 典籍引用 + 内容审核 |

## 八、MVP 极简定义

**MVP = 一个网页，做五件事：**

1. 你输入生日 → 我告诉你是什么底色（3 秒出结果）
2. 你觉得准 → 保存卡片发朋友圈（1 click 裂变）
3. 你想看朋友是什么 → 发链接来合盘（社交裂变）
4. 你想深入了解 → 看专栏报告（AI 生成的深度解读）
5. 你还有问题 → 自由对话（AI 命理分析师实时回答）

**裂变是入口，AI 解读是核心价值。两层缺一不可。**
