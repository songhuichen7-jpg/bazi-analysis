# 合盘体系框架（从第一性原理出发）

> 版本：v1.0 | 2026-04-18 | 状态：**定稿**（10轮审查，终审命理93/传播94/产品93，平均93.3）
> 本文档只定义框架和维度选择，不涉及具体文案内容

---

## 一、第一性原理：合盘到底要回答什么？

### 用户的真实问题

用户做合盘时，心里想的不是"我们的天干五行关系是什么"。他们想知道的是：

1. **我们是什么关系？**（大类定性）
2. **我们在一起会怎样？**（互动模式）
3. **我们的默契在哪里？**（亲密感来源）
4. **我们容易在哪里碰撞？**（潜在摩擦点）
5. **我们各自能给对方什么？**（互补性）

### 产品的双重约束

| 约束 | 具体要求 |
|------|---------|
| 传播性 | 结果要"可截图可发圈"，标签短、金句亮、视觉明确 |
| 命理性 | 懂行的人不该一眼看出"这纯是编的" |
| 正面性 | 所有结果正面重构（产品红线）。**传播学补偿**：不靠"骂人"换唤醒，靠**反差幽默+意外+具体场景**保持高唤醒（参考Berger：高唤醒正面情绪如敬畏/幽默也促进分享） |
| 不限关系 | 不限于恋爱——朋友、同事、搭档都行 |
| 零额外输入 | 只需双方生日，不需要额外问卷 |

## 二、可用信息：从一个生日能算出什么？

用户输入生日（年月日+可选时辰），系统可自动计算：

| 信息 | 来源 | 可靠性 | 说明 |
|------|------|--------|------|
| 日干（天干） | 万年历确定 | 100% | 我们的基础类型核心 |
| 日支 | 万年历确定 | 100% | 贴身位/日常相处位 |
| 月支 | 万年历确定 | 100% | 格局取格的依据 |
| 身强/身弱 | 需四柱综合判断 | ~85% | 已简化为绽放/蓄力 |
| 格局十神 | 日干+月支 | ~90% | 已实现为格局标签 |
| 年柱 | 万年历确定 | 100% | 用于年命/辅助信息 |
| 时柱 | 需精确出生时间 | 60%（很多人不知道） | 可选项 |

### 双人可交叉的信息

当两人都输入生日后，可以计算：

| 交叉维度 | 计算方式 | 典籍支持 |
|---------|---------|---------|
| A日干 × B日干 | 五行生克/天干合 | ★★★★★ |
| A日干 + B日干 → 合化？ | 需看月令环境 | ★★★★★ |
| A日支 × B日支 | 六合/冲/生克/比和 | ★★★★（单造日支论妻为经典；双造日支互参为合婚实务常用，非子平专著单章） |
| A格局十神 × B格局十神 | 十神互配 | ★★★★（《神峰通考》专论） |
| A绽放蓄力 × B绽放蓄力 | 能量状态 | ★★★ |
| A月支 × B月支 | 地支关系 | ★★★ |
| A年柱 × B年柱 | 年命 | ★★（《神峰通考》批评） |

## 三、维度选择：用哪些？不用哪些？为什么？

### 选用维度

#### 维度1：日干五行关系（主轴）

**回答的问题**：我们是什么类型的关系？

**典籍依据**：五行生克是命理最基础的框架，所有典籍一致。

**产品角色**：决定关系大类标签。这是传播面最核心的元素。

**分类方式**：

| 关系 | 五行逻辑 | 标签方向 |
|------|---------|---------|
| 相生 | A生B或B生A | 滋养型 |
| 相克 | A克B或B克A | 火花型 |
| 天干合 | 5对特殊组合 | 默契型（需区分合化/不化） |
| 同天干 | 同一类 | 镜像型 |
| 同五行异阴阳 | 甲乙、丙丁等 | 同频型 |
| 无直接关系 | 间隔五行 | 各自型 |

#### 维度2：天干合的合化/合而不化

**回答的问题**：天干合的两个人，在一起会发生什么化学反应？

**典籍依据**：
- 子平真诠第四章（合化原理）、第五章（合而不合的6种情形）
- 滴天髓·化象：「既合而化，如贞妇配义夫，从一而终……合而不化，勉强之意，必非佳偶。」
- 滴天髓·闲神：「合而不化，谓绊住留连，贪彼忌此。」

**产品化方案**：

完整的合化判断需要四柱全局信息，产品中可简化为：

方案A：**基于月令判断**
- 查化气五行是否得月令生旺 → 倾向合化
- 化气五行被月令克制 → 倾向合而不化
- 这个信息从两人的月支就能判断

方案B：**给出两种解读**
- 同时呈现"如果你们合化"和"如果你们合而不化"
- 让用户自行感受哪个更准

方案C：**不区分，但在描述中包含两面**
- "你们天然吸引，但融合的程度取决于你们各自的状态"

**建议**：**方案B为主，方案A为辅助信号**。

主方案B：并列"融合解读"和"磁铁解读"两种，各1-2句。让用户自己感受哪个更准。
辅助A：用月令是否生旺化气作为**弱倾向信号**（非决定性判断）——在UI上可表现为"更可能偏向XX"的轻引导，但不做断言。
兜底C：若工期紧张，在文案中包含两面性即可。

**命理依据**：《子平真诠》论真假化的条件包括日主根苗、闲神伤化、争合妒合等多端，仅凭月支不足以断定真化。方案B的"并列解读"恰好呼应了典籍中化有真假的学理精神。

#### 维度3：日支关系

**回答的问题**：我们的亲密默契如何？相处中的底层温度是什么？

**典籍依据**：
- 渊海子平：「日为妻妾己身」
- 经云（渊海/三命通会共引）：「日为妻妾己身」；三命通会另有「日坐命财……主得妻财，又主妻贤明」论日支
- 滴天髓·夫妻：「日主坐财，财为喜用者，必得正好妻财」
- 子平真诠·论妻子：「妻以配身……于人愈近，其验益灵」

日支是四柱中「最近」的位置——代表最亲密关系的底色。

**泛关系适配**：日支在传统命理中称"夫妻宫"，严格来说针对婚恋。但子平学理的底层逻辑是"日元坐下为贴身、切近"——描述的是**日常相处中最自然的互动模式**。产品中改称"贴身位/相处底盘"，用于所有关系类型（朋友、搭档、同事均适用）。若用户在AI对话中明确问婚恋，AI可补充传统夫妻宫解读。

**分类方式**：

A的日支与B的日支之间的关系：

| 关系 | 包含 | 含义 |
|------|------|------|
| 六合 | 子丑、寅亥、卯戌、辰酉、巳申、午未 | 易和、易商量、配合度高（非无条件吉——六合在喜忌不利时也可为羁绊） |
| 相冲 | 子午、丑未、寅申、卯酉、辰戌、巳亥 | 底层张力，需要磨合 |
| 相生 | 按五行 | 自然滋养 |
| 相克 | 按五行 | 有摩擦的成长 |
| 比和 | 同支或同五行 | 天然同频 |
| 其他 | 无特殊关系 | 各自独立 |

**注意**：六合和相冲是日支维度中最有戏剧性的关系，应该给予最强的表达。其他关系作为轻度修饰。

#### 维度4：十神互配（格局标签之间的关系）

**回答的问题**：我们的能量是互补的还是同类的？

**典籍依据**：
- 《神峰通考·男女合婚说》：「比肩劫财重者，必择妇命带伤官食神重者配之。此系合婚之正理。」
- 五行相生链：比劫→食伤→财→官杀→印枭→比劫

《神峰通考》的"比劫配食伤"是**唯一明文的双造互配法则**。以下是基于五行流通逻辑的推广应用（非原文全十神合婚条例）。

**分类方式**：

基于双方格局标签中的十神，按五行相生链判断：

| A的十神 | B的十神 | 五行关系 | 互配含义 |
|---------|---------|---------|---------|
| 比劫类 | 食伤类 | A→B（底气→出口） | 你的底气是TA表达的出口 |
| 食伤类 | 财星类 | A生B | 你的才华通过TA变现 |
| 财星类 | 官杀类 | A生B | 你的资源支撑TA的抱负 |
| 官杀类 | 印枭类 | A生B | 你的压力转化为TA的学习 |
| 印枭类 | 比劫类 | A生B | 你的知识增强TA的力量 |
| 同类 | 同类 | 比和 | 同频共振，不用解释 |
| 相克 | 相克 | A克B或B克A | 有角色拉扯的互动（一方约束/一方表达，制衡而非对抗）。**注**：此"克"为十神名目归五行后的流通关系，非双方日干直接相战（层次1已论日干五行克） |

**简化版**（十神归入五行）：

| 十神 | 归属五行 |
|------|---------|
| 比肩、劫财 | 与日干同行 |
| 食神、伤官 | 日干所生 |
| 正财、偏财 | 日干所克 |
| 正官、七杀 | 克日干者 |
| 正印、偏印 | 生日干者 |

双方十神之间的五行关系 = 十神互配关系

#### 维度5：绽放/蓄力动态状态（已有）

**回答的问题**：我们各自的能量状态如何？

已在04b中实现。保留。

### 不选用的维度及原因

| 维度 | 不选原因 |
|------|---------|
| 纳音 | 用户明确排除；滴天髓批评「纳音诸法，尤属不经」 |
| 地支刑/害/破 | 任铁樵：「刑之义无所取……害之义，尤为穿凿」 |
| 地支三合/三会 | 双人也可论三合（需有第三支介入或本命自带），但实现复杂度高且与日支层次部分重叠。排除理由是**解释成本与信息增益不成比例**，非"不能用于双人"。 |
| 年柱比对 | 《神峰通考》批评的是**仅以年命合婚**（吕才法），非"年柱毫无信息"。但年柱在双人互参中信息密度低于日柱，且与日干主轴部分重叠，MVP阶段不纳入。 |
| 神煞（桃花等） | 任铁樵：「桃花咸池……神煞纳音诸名目，乃好事妄造」 |
| 完整四柱互参 | 需要精确出生时间（多数用户不知道）；复杂度过高 |
| 大运/流年 | 留给核心产品层的"年度关系运势"报告 |
| 月支地支关系 | 与日支相比优先级低；宫位远近原则（日主最近） |

## 四、维度叠加架构

### 核心原则：传播面 vs 计算层分离

用户截图分享的是**传播面**（标签+金句），不是计算过程。分享卡只卖**1个记忆点+1个情绪钩子**，其余维度进详情页/AI对话。

```
传播面（分享卡上可见的）
┌──────────────────────────────────────────┐
│  主标签（层次1，≤4字）   ← 唯一记忆单元   │
│  金句（1句）            ← 唯一情绪钩子    │
│  状态图标（层次5）      ← 视觉辅助        │
└──────────────────────────────────────────┘

计算层（决定主标签和金句内容的）
┌──────────────────────────────────────────┐
│  层次1：关系大类（天干五行）               │
│  层次2：化学反应（合化/不化）[天干合限定]   │
│  层次3：底层温度（日支关系）               │
│  层次4：能量互补（十神五行流通）           │
│  层次5：动态状态（绽放/蓄力）              │
└──────────────────────────────────────────┘

详情层（下滑可见，与00_产品架构的卡片式下滑一致）
┌──────────────────────────────────────────┐
│  副标签（层次3：底层温度，≤6字）           │
│  互补描述（层次4：1句）                    │
│  合化双解读（层次2：仅天干合时）           │
│  相处建议（综合5层生成）                   │
└──────────────────────────────────────────┘
```

### 信息分层

```
层次1：关系大类（天干五行）        → "你们是火花搭子"
层次2：化学反应（合化/不化）        → "你们天然吸引——融合型or磁铁型？"  [仅限天干合]
层次3：底层温度（日支关系）         → "底层默契很强"
层次4：能量互补（十神五行流通）     → "你的创意TA能接住"
层次5：动态状态（绽放/蓄力）        → "一个在冲，一个在稳"
```

### 每个层次的输出

| 层次 | 输出元素 | 传播面/详情层 | 呈现方式 |
|------|---------|-------------|---------|
| 层次1 | 关系大类标签（≤4字）| **传播面** | 大字，卡片最醒目位置 |
| 层次2 | 化学反应双解读 | **详情层** | 仅天干合时出现，展开可见 |
| 层次3 | 底层温度标签（≤6字）| **详情层** | 副标签，展开/下滑可见 |
| 层次4 | 互补描述（1句）| **详情层** | 融入详情段落 |
| 层次5 | 状态标识（⚡🔋）| **传播面** | 小图标，分享卡可见 |

### MVP分阶

|------|---------|---------|--------|

### 合盘结果页三屏信息架构

| 屏序 | 职责 | 内容 | 信息量控制 |
|------|------|------|-----------|
| 首屏（传播面） | 唯一记忆点+唯一情绪钩子 | 主标签（≤4字）、金句（≤28字）、双状态图标、极简双身份（一行图标化）、品牌水印、分享CTA（短链为主） | **记忆单元≤3**（仅计语义块：主标签+金句+状态对比）；水印/CTA/双身份为**界面噪声预算**，不计入记忆单元 |
| 第二屏（详情层） | 深度叙事 | 副标签"底层温度"、十神互补1句、合化双解读（天干合限定）、1句可执行相处建议 | ≤4个模块，总文字≤200字 |
| 第三屏（转化层） | 回流核心产品 | "想问具体场景？"→AI预设问句；"生成相处指南"→专栏；邀请对方/保存 | 1个主CTA+≤3个次级入口 |

### 合盘回流路径（受邀方）

对方收到分享卡→点进来的路径需要专项设计（不同于自发测试路径）：

1. **落地首屏（模糊预览）**：显示"你与[昵称/类型名]的合盘预览"——
   - **模糊预览规格**：可见关系大类标签，金句部分用精致纹样遮罩（非裸"???"），保持视觉品质
   - **动机句**固定句式："输入你的生日，解锁你们的同框解读"（受益陈述优先于权限陈述）
   - **悬念伦理红线**：遮挡的是文案内容而非关系尊严；禁止羞辱/胁迫/猎奇向预览（如"你们的关系有个大问题"）；禁止看起来像空白页/错误页/廉价弹窗
   - **双身份展示规格**：字段优先级为 昵称>类型昵称>默认代称"TA"；脱敏规则与首屏一致
2. **解锁条件**：输入自己的生日（对方信息已由邀请方锁定在invite_id中，无需重复输入）
3. **解锁后**：直接进双人合盘结果（不先展示两轮单人卡片），单人卡片作为次级展开
4. **分享卡CTA**：卡片上需有可读的产品入口线索（优先级：短链>口令>小程序搜索），否则截图在朋友圈不可点击时链路会断

### 合盘邀请安全与隐私

- **高熵token**：invite_id使用不可枚举的随机字符串（≥128bit），HTTPS传输
- **可见性分层**：仅凭链接可见"关系大类梗+模糊预览"；发起人完整排盘信息仅在受邀方提交生日后可见
- **一次性绑定**：一个invite_id只允许一个受邀方完成解锁，防止多人共用同一邀请
- **失效与撤销**：invite_id设合理有效期；发起人可主动使链接失效；发起人改生日后旧invite作废并提示受邀方"对方信息已更新"
- **隐私声明**：受邀方提交生日即建立合盘关系，落地页需一行说明生日用途与可见范围
- **降级策略**：invite失效/异常时展示"邀请已过期，邀请TA重新发一个"，不展示错误页

### 组合数学

| 层次 | 分类数 | 说明 |
|------|--------|------|
| 层次1 | 6 | 天作/滋养/火花/镜像/同频/各自 |
| 层次2 | 3 | 合化/不化/不适用 |
| 层次3 | 6 | 六合/冲/生/克/比和/无关 |
| 层次4 | 7 | 5种相生+比和+相克 |
| 层次5 | 4 | 双绽/绽蓄/蓄绽/双蓄 |

理论最大组合：6 × 3 × 6 × 7 × 4 = **3,024种**

实际有效组合更少（因为天干合只有5对，很多层次组合不会同时出现），但足够保证每对用户的结果都有独特感。

### 数据契约：合盘复用单人排盘

合盘API应**复用单人排盘结果**而非重复计算，确保：
- 合盘卡片上的天干/日支/格局标签/绽放蓄力与**单人卡片完全一致**
- 用户在单人卡片中已看到的信息不会在合盘中出现不一致版本
- 缺时辰时：单人格局标签仍可算出（依赖日干+月支），合盘层次4可正常工作；但应在UI上标注"基于出生年月日，未含时辰"

**边界情况**：

| 场景 | 处理策略 |
|------|---------|
| 双方同年月日生日 | `chart_a_id == chart_b_id`时进入专项模板分支："同盘不同境"叙事，强调角色分工与场景差异而非重复单人结论 |
| A/B顺序对调 | `pair_key = sort(person_a, person_b)`规范化；文案中"你/TA"锚定发起方视角 |
| 一方改生日/重算 | 分享URL带`paipan_hash`；打开时提示"对方资料已更新，可刷新" |
| 一人有时辰、一人无 | 合盘只用双方"已共识的排盘快照"，不因一方有时辰而升级另一方的维度 |
| 历法边界（立春换年等） | 统一使用排盘引擎的公历→农历转换规则，不做真太阳时校正（MVP阶段） |
| 时区与日界 | 统一以东八区(UTC+8)午夜为日界，与排盘引擎一致；跨时区用户按设备时区转换后的东八区日期输入 |
| 无效/异常输入 | 复用单人排盘同一校验规则（非法日期、未来日期等）；合盘不做特例 |
